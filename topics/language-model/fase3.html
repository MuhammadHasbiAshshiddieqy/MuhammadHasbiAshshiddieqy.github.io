3.1 Mekanisme Attention // dari bottleneck ke alignment

Attention mechanism lahir dari satu pertanyaan sederhana: alih-alih memaksa encoder merangkum seluruh input menjadi satu vektor, mengapa tidak biarkan decoder "melihat" semua output encoder dan memilih mana yang relevan di setiap langkah decoding? Intuisi ini, yang diperkenalkan oleh Bahdanau et al. (2014), secara dramatis meningkatkan kualitas machine translation dan menjadi fondasi semua model bahasa modern.

Mekanisme attention secara esensial adalah weighted average: setiap hidden state encoder mendapat bobot (alignment score) berdasarkan relevansinya dengan hidden state decoder saat ini. Bobot-bobot ini dijumlahkan secara softmax sehingga total bobot = 1, menghasilkan sebuah context vector yang dikomputasi secara dinamis untuk setiap langkah decoding — berbeda dengan context vector statis di Seq2Seq klasik.

Dua jenis attention yang fundamental

  • Additive Attention (Bahdanau) — alignment score dihitung dengan MLP kecil: score(h_s, h_t) = v^T · tanh(W_1·h_s + W_2·h_t). Lebih fleksibel karena ada parameter yang dipelajari, tapi lebih lambat karena operasi sequential.
  • Multiplicative/Dot-Product Attention (Luong) — alignment score dihitung dengan dot product: score(h_s, h_t) = h_s^T · h_t. Lebih cepat secara komputasi (bisa diparalelkan sebagai matrix multiplication), menjadi basis dari Transformer's scaled dot-product attention.
Pythonbahdanau_attention.py
import torch
import torch.nn as nn
import torch.nn.functional as F

class BahdanauAttention(nn.Module):
    """Additive attention — query dari decoder, keys/values dari encoder outputs."""
    def __init__(self, hidden_size: int):
        super().__init__()
        self.W1 = nn.Linear(hidden_size, hidden_size, bias=False)  # transform encoder states
        self.W2 = nn.Linear(hidden_size, hidden_size, bias=False)  # transform decoder state
        self.v = nn.Linear(hidden_size, 1, bias=False)             # project to scalar score

    def forward(self, query, keys, values, mask=None):
        """
        query:  [batch, hidden]         — decoder hidden state saat ini
        keys:   [batch, src_len, hidden] — semua encoder hidden states
        values: [batch, src_len, hidden] — sama dengan keys (biasanya)
        """
        query = query.unsqueeze(1)  # [batch, 1, hidden] untuk broadcasting

        # Alignment scores: tanh(W1·keys + W2·query)
        energy = torch.tanh(self.W1(keys) + self.W2(query))  # [batch, src_len, hidden]
        scores = self.v(energy).squeeze(-1)                   # [batch, src_len]

        # Masking: abaikan padding tokens
        if mask is not None:
            scores = scores.masked_fill(mask == 0, -1e9)

        # Softmax → attention weights (sum = 1)
        weights = F.softmax(scores, dim=-1)  # [batch, src_len]

        # Context vector: weighted sum of values
        context = torch.bmm(weights.unsqueeze(1), values)  # [batch, 1, hidden]
        context = context.squeeze(1)                       # [batch, hidden]

        return context, weights  # weights bisa divisualisasikan sebagai alignment matrix!

3.2 Self-Attention & Scaled Dot-Product // setiap token melihat semua token

Self-attention adalah variasi attention di mana query, key, dan value semuanya berasal dari sequence yang sama. Alih-alih decoder memperhatikan encoder (cross-attention), setiap token dalam sequence memperhatikan semua token lainnya — termasuk dirinya sendiri. Ini memungkinkan model menangkap dependensi jarak jauh dalam satu langkah komputasi, tanpa memerlukan RNN sequential.

Konsep Query-Key-Value (QKV) adalah abstraksi yang sangat berguna: bayangkan kamu mencari dokumen di database. Query adalah pertanyaanmu, Key adalah indeks dokumen-dokumen yang tersedia, dan Value adalah konten aktual dokumen. Attention score = seberapa cocok query dengan setiap key. Output = rata-rata tertimbang dari values, berbobot sesuai attention scores.

Pythonself_attention.py
import torch
import torch.nn as nn
import torch.nn.functional as F
import math

def scaled_dot_product_attention(
    Q: torch.Tensor,   # [batch, heads, seq, d_k] — Queries
    K: torch.Tensor,   # [batch, heads, seq, d_k] — Keys
    V: torch.Tensor,   # [batch, heads, seq, d_v] — Values
    mask: torch.Tensor = None,
) -> tuple[torch.Tensor, torch.Tensor]:
    d_k = Q.size(-1)  # dimensi key

    # Step 1: Attention scores = Q · K^T / sqrt(d_k)
    # Scaling dengan sqrt(d_k) mencegah dot products terlalu besar
    # yang menyebabkan softmax menghasilkan gradient yang sangat kecil
    scores = torch.matmul(Q, K.transpose(-2, -1)) / math.sqrt(d_k)
    # scores: [batch, heads, seq_q, seq_k]

    # Step 2: Masking (untuk causal LM: token tidak boleh lihat masa depan)
    if mask is not None:
        scores = scores.masked_fill(mask == 0, -1e9)  # set ke -inf → softmax ≈ 0

    # Step 3: Softmax → attention weights
    weights = F.softmax(scores, dim=-1)  # [batch, heads, seq_q, seq_k]

    # Step 4: Weighted sum of values
    output = torch.matmul(weights, V)  # [batch, heads, seq_q, d_v]

    return output, weights

class SingleHeadSelfAttention(nn.Module):
    def __init__(self, d_model: int):
        super().__init__()
        self.d_model = d_model
        # Learnable projections: input → Q, K, V
        self.W_q = nn.Linear(d_model, d_model, bias=False)
        self.W_k = nn.Linear(d_model, d_model, bias=False)
        self.W_v = nn.Linear(d_model, d_model, bias=False)
        self.W_o = nn.Linear(d_model, d_model)  # output projection

    def forward(self, x, mask=None):
        # x: [batch, seq_len, d_model]
        Q = self.W_q(x)  # [batch, seq, d_model]
        K = self.W_k(x)  # semua dari x yang sama → self-attention
        V = self.W_v(x)

        Q = Q.unsqueeze(1)  # add head dim untuk compatibility
        K = K.unsqueeze(1)
        V = V.unsqueeze(1)

        attn_output, weights = scaled_dot_product_attention(Q, K, V, mask)
        attn_output = attn_output.squeeze(1)  # [batch, seq, d_model]
        return self.W_o(attn_output), weights
O(n²) kompleksitas adalah bottleneck nyataSelf-attention menghitung attention score antara setiap pasangan token: untuk sequence panjang n, ini O(n²) dalam waktu dan memori. Untuk n=1000 ini fine, tapi n=32.000 (GPT-4 extended) atau n=1.000.000 (Gemini Ultra) membutuhkan teknik khusus: FlashAttention (I/O-aware recomputation), Sparse Attention, Linear Attention, atau approximate methods. Ini adalah area riset aktif hingga 2025.

3.3 Multi-Head Attention & Positional Encoding // melihat dari banyak sudut

Single-head attention hanya bisa memperhatikan satu jenis hubungan antar token pada satu waktu. Multi-Head Attention (MHA) menjalankan beberapa attention heads secara paralel, masing-masing dengan projection matrix W_Q, W_K, W_V yang berbeda — memungkinkan model menangkap berbagai jenis hubungan secara simultan: satu head mungkin menangkap hubungan sintaktik, head lain menangkap koreferensi, head lain menangkap hubungan semantik.

Positional Encoding mengatasi kelemahan fundamental self-attention: tidak punya built-in sense of order. Tanpa PE, "the cat sat" dan "sat cat the" menghasilkan output identik karena attention tidak peduli urutan — hanya siapa memperhatikan siapa. PE menyuntikkan informasi posisi ke dalam representasi token, memungkinkan model membedakan urutan.

Pythonmulti_head_attention.py
import torch
import torch.nn as nn
import torch.nn.functional as F
import math

class MultiHeadAttention(nn.Module):
    def __init__(self, d_model: int = 512, num_heads: int = 8):
        super().__init__()
        assert d_model % num_heads == 0, "d_model harus habis dibagi num_heads"

        self.d_model = d_model
        self.num_heads = num_heads
        self.d_k = d_model // num_heads  # dimensi per head

        # Single projection matrix untuk semua heads (lebih efisien)
        self.W_q = nn.Linear(d_model, d_model, bias=False)
        self.W_k = nn.Linear(d_model, d_model, bias=False)
        self.W_v = nn.Linear(d_model, d_model, bias=False)
        self.W_o = nn.Linear(d_model, d_model)

    def forward(self, query, key, value, mask=None):
        batch_size = query.size(0)

        # Project dan split ke multiple heads
        def split_heads(x):
            # x: [batch, seq, d_model] → [batch, heads, seq, d_k]
            x = x.view(batch_size, -1, self.num_heads, self.d_k)
            return x.transpose(1, 2)

        Q = split_heads(self.W_q(query))  # [batch, heads, seq_q, d_k]
        K = split_heads(self.W_k(key))
        V = split_heads(self.W_v(value))

        # Scaled dot-product attention (paralel untuk semua heads)
        scores = torch.matmul(Q, K.transpose(-2, -1)) / math.sqrt(self.d_k)
        if mask is not None:
            scores = scores.masked_fill(mask == 0, -1e9)
        weights = F.softmax(scores, dim=-1)
        attn = torch.matmul(weights, V)  # [batch, heads, seq, d_k]

        # Concatenate heads → [batch, seq, d_model]
        attn = attn.transpose(1, 2).contiguous()
        attn = attn.view(batch_size, -1, self.d_model)

        return self.W_o(attn)

class SinusoidalPositionalEncoding(nn.Module):
    """Original Transformer PE — deterministik, tidak perlu dilatih."""
    def __init__(self, d_model: int, max_len: int = 5000, dropout: float = 0.1):
        super().__init__()
        self.dropout = nn.Dropout(dropout)

        pe = torch.zeros(max_len, d_model)
        position = torch.arange(0, max_len).unsqueeze(1).float()
        div_term = torch.exp(
            torch.arange(0, d_model, 2).float() * (-math.log(10000.0) / d_model)
        )
        # Sinusoidal: posisi genap = sin, posisi ganjil = cos
        pe[:, 0::2] = torch.sin(position * div_term)
        pe[:, 1::2] = torch.cos(position * div_term)
        self.register_buffer("pe", pe.unsqueeze(0))  # [1, max_len, d_model]

    def forward(self, x):
        # x: [batch, seq_len, d_model]
        x = x + self.pe[:, :x.size(1)]
        return self.dropout(x)

# Modern alternative: RoPE (Rotary Position Embedding) — digunakan LLaMA, GPT-NeoX
# RoPE encode posisi sebagai rotasi dalam ruang complex — extrapolate ke panjang yang lebih panjang
Sinusoidal PE vs Learned PE vs RoPESinusoidal PE dari paper asli Transformer: deterministik, bisa extrapolate ke sequence lebih panjang dari training. Learned PE: dilatih dari data, mungkin lebih baik untuk panjang yang dilihat saat training, tapi tidak bisa extrapolate. RoPE (Rotary): encoding relatif berbasis rotasi, generalizes lebih baik dan menjadi standard di LLM modern (LLaMA, Mistral, Qwen). ALiBi: alternatif berbasis position bias tanpa embedding eksplisit.

3.4 Arsitektur Transformer Lengkap // "Attention is All You Need"

Paper "Attention is All You Need" (Vaswani et al., 2017) mengusulkan arsitektur yang sepenuhnya menghilangkan rekurensi dan konvolusi — hanya menggunakan attention dan feedforward layers. Hasilnya lebih paralel (training lebih cepat), lebih powerful (menangkap dependensi jarak jauh lebih baik), dan menjadi template untuk semua LLM modern.

Arsitektur full Transformer terdiri dari dua komponen utama: Encoder Stack yang membangun representasi input dengan full self-attention (setiap token bisa memperhatikan semua token lain), dan Decoder Stack yang menghasilkan output secara autoregresif dengan causal self-attention (hanya bisa memperhatikan token sebelumnya) ditambah cross-attention ke output encoder.

Pythontransformer_block.py
import torch
import torch.nn as nn

class TransformerEncoderLayer(nn.Module):
    """Satu encoder block: MHA → Add&Norm → FFN → Add&Norm."""
    def __init__(self, d_model: int = 512, num_heads: int = 8,
                 d_ff: int = 2048, dropout: float = 0.1):
        super().__init__()
        # Multi-Head Self-Attention
        self.self_attn = MultiHeadAttention(d_model, num_heads)
        self.norm1 = nn.LayerNorm(d_model)

        # Position-wise Feed-Forward Network (FFN)
        # d_ff biasanya = 4 × d_model — expansion dan contraction
        self.ffn = nn.Sequential(
            nn.Linear(d_model, d_ff),
            nn.GELU(),
            nn.Dropout(dropout),
            nn.Linear(d_ff, d_model),
        )
        self.norm2 = nn.LayerNorm(d_model)
        self.dropout = nn.Dropout(dropout)

    def forward(self, x, mask=None):
        # Pre-LN vs Post-LN: Pre-LN (modern, lebih stabil):
        # x = x + dropout(MHA(LN(x)))  ← norm SEBELUM sublayer
        # Post-LN (original Transformer): x = LN(x + dropout(MHA(x)))

        # Multi-Head Self-Attention + Residual Connection
        attn_output = self.self_attn(self.norm1(x), self.norm1(x), self.norm1(x), mask)
        x = x + self.dropout(attn_output)  # Residual connection

        # FFN + Residual Connection
        x = x + self.dropout(self.ffn(self.norm2(x)))

        return x

class TransformerDecoderLayer(nn.Module):
    """Decoder block: Masked MHA → Cross-Attention → FFN."""
    def __init__(self, d_model=512, num_heads=8, d_ff=2048, dropout=0.1):
        super().__init__()
        self.self_attn = MultiHeadAttention(d_model, num_heads)
        self.cross_attn = MultiHeadAttention(d_model, num_heads)  # attend to encoder
        self.ffn = nn.Sequential(nn.Linear(d_model, d_ff), nn.GELU(), nn.Linear(d_ff, d_model))
        self.norm1 = nn.LayerNorm(d_model)
        self.norm2 = nn.LayerNorm(d_model)
        self.norm3 = nn.LayerNorm(d_model)
        self.dropout = nn.Dropout(dropout)

    def forward(self, x, enc_output, src_mask=None, tgt_mask=None):
        # 1. Masked Self-Attention (causal — decoder token tidak lihat masa depan)
        x = x + self.dropout(self.self_attn(self.norm1(x), self.norm1(x), self.norm1(x), tgt_mask))

        # 2. Cross-Attention: query dari decoder, key/value dari encoder output
        x = x + self.dropout(self.cross_attn(self.norm2(x), enc_output, enc_output, src_mask))

        # 3. FFN
        x = x + self.dropout(self.ffn(self.norm3(x)))
        return x

3.5 Varian Transformer // encoder, decoder, dan encoder-decoder

Setelah paper "Attention is All You Need", komunitas riset dengan cepat menyadari bahwa berbagai task NLP bisa memanfaatkan bagian berbeda dari arsitektur Transformer. Tidak semua task butuh encoder-decoder penuh — dan pilihan arsitektur yang tepat memiliki implikasi besar terhadap cara model dilatih, cara digunakan, dan kapabilitas yang dimiliki.

Tiga keluarga besar varian Transformer telah mendominasi NLP modern: encoder-only (BERT dan keluarganya) yang menyandikan teks secara bidirectional untuk understanding tasks, decoder-only (GPT dan keluarganya) yang menghasilkan teks secara autoregresif dan menjadi backbone LLM modern, serta encoder-decoder (T5, BART) yang mempertahankan kedua komponen untuk translation dan conditional generation.

Encoder-Only (BERT style)

Self-attention: bidirectional — setiap token bisa memperhatikan semua token lain. Dilatih dengan Masked Language Modeling (MLM): prediksi token yang di-mask. Kekuatan: representasi kontekstual sangat kaya untuk understanding. Lemah: tidak dirancang untuk text generation. Contoh: BERT, RoBERTa, DeBERTa, ELECTRA.

Decoder-Only (GPT style)

Self-attention: causal (autoregressive) — hanya bisa lihat token sebelumnya. Dilatih dengan causal language modeling: prediksi token berikutnya. Kekuatan: text generation natural, in-context learning powerful, satu model untuk banyak task tanpa fine-tuning. Contoh: GPT series, LLaMA, Mistral, Gemini, Claude.

Encoder-Decoder (T5 style)

Encoder: bidirectional, Decoder: causal + cross-attention. Dilatih dengan denoising objectives. Kekuatan: sangat baik untuk conditional generation (translation, summarization, Q&A) karena encoder membangun representasi input yang kaya. Contoh: T5, BART, mBART, Flan-T5.

Kapan memilih mana?

Classification, NER, extractive QA → Encoder-only. Text generation, chat, code → Decoder-only. Translation, summarization, structured generation → Encoder-decoder. Di 2024, decoder-only model besar sering mengalahkan encoder-decoder untuk semua task berkat scale dan RLHF.

Mengapa decoder-only mendominasi 2023–2025?Skalabilitas: decoder-only lebih mudah di-scale karena tidak ada asimetri encoder/decoder. Fleksibilitas: satu model untuk semua task via prompting, tanpa fine-tuning per task. In-context learning: LLM skala besar menunjukkan kemampuan few-shot yang kuat dari decoder-only training. RLHF juga lebih natural untuk decoder-only karena evaluasi berbasis generation.
