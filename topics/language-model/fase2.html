2.1 Fondasi Neural Network // bangunan dasar

Neural network adalah fungsi matematika yang dipelajari dari data. Memahami mekanisme dasarnya — perceptron, aktivasi, forward pass, dan backpropagation — bukan hanya formalitas akademik. Ketika model kamu tidak konvergen, atau ketika kamu perlu memilih optimizer dan learning rate yang tepat, intuisi tentang cara training bekerja akan sangat menentukan.

Konsep kunci yang sering disalahpahami: neural network tidak "menemukan aturan" secara eksplisit. Ia mengoptimasi sekumpulan parameter numerik (bobot) sehingga fungsi yang dipetakan oleh jaringan tersebut memiliki output yang mendekati target pada data training. Seluruh "kecerdasan" tersimpan dalam ratusan juta angka floating-point — dan backpropagation adalah algoritma yang mengupdate angka-angka tersebut secara efisien.

Dari Persamaan Garis ke Neural Network — Bangun Intuisi

Neural network dimulai dari persamaan garis: y = w·x + b. Geser slider untuk lihat bagaimana weight (w) mengubah kemiringan dan bias (b) menggeser garis.

y = 1.0·x + 0.0
w → kemiringan garis
b → geser naik/turun
Satu neuron = satu persamaan garis

Sebuah neuron menerima beberapa input, masing-masing dikali weight-nya, dijumlah, ditambah bias. Geser x₁ dan x₂ untuk lihat bagaimana output berubah.

z = w₁x₁ + w₂x₂ + b
w₁=0.8, w₂=0.5, b=0.2
z = weighted sum of inputs
output z = 1.50

Tanpa aktivasi, jaringan sebesar apapun hanya bisa belajar fungsi linear. Fungsi aktivasi menambahkan non-linearitas — inilah yang membuat neural network bisa memahami pola kompleks.

ReLU: max(0, x)
Output negatif → 0
Output positif → identik

Sederhana, cepat, paling umum

Susun neuron-neuron menjadi layer. Output satu layer jadi input layer berikutnya. Geser input untuk lihat bagaimana sinyal mengalir dari kiri ke kanan.

Input → Hidden (ReLU) → Output
Output: –
Warna node = aktivasi
Tebal edge = bobot koneksi
Forward Pass — Dari Input ke Prediksi
Input x
Teks → token IDs → embedding vectors [batch, seq_len, d_model]
Linear Transform
z = W·x + b — weighted sum of inputs
Activation Function
a = ReLU(z) atau σ(z) — tambahkan non-linearitas
Loss Computation
L = CrossEntropy(output, target) — seberapa salah prediksi?
Backpropagation
∂L/∂W — chain rule, update W -= lr × gradient
↺ Ulangi untuk setiap batch hingga loss konvergen

Activation functions yang sering digunakan di NLP

  • ReLU (Rectified Linear Unit) — max(0, x). Menghindari vanishing gradient, komputasi cepat. Masalah: "dying ReLU" jika neuron selalu negatif → gradiennya nol permanen.
  • GELU (Gaussian Error Linear Unit) — digunakan di BERT dan GPT. Smooth approximation dari ReLU, empirically lebih baik untuk transformer.
  • Sigmoid — 1/(1+e^-x), output 0–1. Cocok untuk output layer klasifikasi biner. Di hidden layer sudah jarang karena saturasi dan vanishing gradient.
  • Softmax — normalisasi vektor menjadi distribusi probabilitas (sum=1). Digunakan di output layer LM untuk mendistribusikan probabilitas di atas vocabulary.
Pythonneural_net_basics.py
import torch
import torch.nn as nn
import torch.optim as optim

# Simple feedforward network untuk klasifikasi teks
class TextClassifier(nn.Module):
    def __init__(self, vocab_size: int, embed_dim: int, num_classes: int):
        super().__init__()
        # Embedding: vocab_size × embed_dim
        self.embedding = nn.Embedding(vocab_size, embed_dim, padding_idx=0)
        self.network = nn.Sequential(
            nn.Linear(embed_dim, 256),
            nn.GELU(),           # smooth, modern activation
            nn.Dropout(0.3),   # regularization
            nn.Linear(256, 64),
            nn.GELU(),
            nn.Linear(64, num_classes)
        )

    def forward(self, token_ids: torch.Tensor) -> torch.Tensor:
        # token_ids: [batch, seq_len]
        embedded = self.embedding(token_ids)      # [batch, seq_len, embed_dim]
        pooled = embedded.mean(dim=1)              # [batch, embed_dim] — mean pooling
        logits = self.network(pooled)             # [batch, num_classes]
        return logits

# Training loop minimal
model = TextClassifier(vocab_size=10000, embed_dim=128, num_classes=5)
optimizer = optim.AdamW(model.parameters(), lr=3e-4, weight_decay=0.01)
criterion = nn.CrossEntropyLoss()

for epoch in range(10):

AdamW, bukan AdamUntuk NLP tasks modern, selalu gunakan AdamW (Adam with decoupled weight decay), bukan Adam. Weight decay di AdamW diaplikasikan langsung ke weights, bukan ke gradients — ini lebih benar secara matematis dan menghasilkan generalisasi lebih baik. Juga: zero_grad(set_to_none=True) lebih efisien dari zero_grad().

2.2 CNN untuk Teks // local patterns sebagai fitur

Convolutional Neural Network (CNN) dikenal karena computer vision, tapi CNN 1D sangat efektif untuk teks — terutama untuk klasifikasi. Intuisinya menarik: filter konvolusi dalam konteks teks berperan sebagai detektor n-gram yang dipelajari secara otomatis. Filter lebar 3 mendeteksi trigram features, filter lebar 4 mendeteksi 4-gram, dan seterusnya.

Kekuatan utama CNN untuk teks: parallelism dan efisiensi. Tidak seperti RNN yang memproses token secara sekuensial, CNN memproses semua posisi secara bersamaan — training jauh lebih cepat. Untuk klasifikasi teks (sentimen, topik, spam detection), CNN seringkali match atau bahkan overtake RNN dengan training time yang jauh lebih singkat.

Pythontext_cnn.py
import torch
import torch.nn as nn

class TextCNN(nn.Module):
    """TextCNN — Kim (2014): multiple filter sizes sebagai n-gram detectors."""
    def __init__(self, vocab_size, embed_dim=128, num_filters=100, num_classes=2):
        super().__init__()
        self.embedding = nn.Embedding(vocab_size, embed_dim)

        # Filter dengan berbagai ukuran — mendeteksi n-gram berbeda
        self.convs = nn.ModuleList([
            nn.Conv1d(
                in_channels=embed_dim,
                out_channels=num_filters,
                kernel_size=k  # k=2: bigram, k=3: trigram, k=4: 4-gram
            )
            for k in [2, 3, 4]
        ])

        self.dropout = nn.Dropout(0.5)
        self.fc = nn.Linear(num_filters * 3, num_classes)  # 3 filter sizes

    def forward(self, x):
        # x: [batch, seq_len]
        embedded = self.embedding(x)          # [batch, seq_len, embed_dim]
        embedded = embedded.permute(0, 2, 1) # [batch, embed_dim, seq_len] — Conv1d format

        pooled_outputs = []
        for conv in self.convs:
            activated = torch.nn.functional.relu(conv(embedded))  # [batch, filters, seq-k+1]
            pooled = activated.max(dim=2).values                   # max-over-time pooling
            pooled_outputs.append(pooled)

        # Concatenate features dari semua filter sizes
        features = torch.cat(pooled_outputs, dim=1)  # [batch, num_filters*3]
        features = self.dropout(features)
        return self.fc(features)                       # [batch, num_classes]

# Key insight: max-over-time pooling memilih feature terkuat dari setiap filter
# — seolah bertanya "apakah trigram ini PERNAH muncul di dokumen ini?"
# Output panjang dokumen tidak mempengaruhi ukuran representasi akhir.
Kapan pakai CNN vs RNN untuk teksPilih CNN ketika: task adalah klasifikasi (bukan generation), kamu butuh training cepat, panjang dokumen bervariasi jauh. Pilih RNN/LSTM ketika: order token sangat penting, task membutuhkan understanding konteks sekuensial (time series, speech), atau kamu butuh generate teks token by token. Di 2024+, keduanya sebagian besar digantikan oleh Transformer — tapi untuk edge deployment atau resource-constrained environment, CNN masih relevan.

2.3 RNN & Sequence Modeling // memori dalam urutan

Recurrent Neural Network (RNN) didesain untuk data sekuensial: hidden state dari langkah sebelumnya diumpankan ke langkah berikutnya, sehingga jaringan secara teori "mengingat" konteks masa lalu. Ini adalah lompatan konseptual besar dari feedforward network yang memperlakukan setiap input sebagai independen.

Masalah terbesar RNN — yang memotivasi lahirnya LSTM, GRU, dan akhirnya Attention — adalah vanishing gradient. Saat training dengan backpropagation through time (BPTT), gradien harus dipropagasi melewati ratusan langkah. Karena gradien dikali dengan weight matrix di setiap langkah, jika weight matrix eigenvalue < 1, gradien menyusut secara eksponensial dan dependensi jarak jauh tidak bisa dipelajari. Inilah alasan kenapa RNN dasar hanya efektif untuk konteks pendek.

Pythonrnn_language_model.py
import torch
import torch.nn as nn

class VanillaRNN(nn.Module):
    """RNN manual untuk memahami mekanismenya."""
    def __init__(self, input_size: int, hidden_size: int):
        super().__init__()
        self.hidden_size = hidden_size
        # Weight untuk input dan hidden state
        self.W_xh = nn.Linear(input_size, hidden_size, bias=False)
        self.W_hh = nn.Linear(hidden_size, hidden_size)  # recurrent connection
        self.W_hy = nn.Linear(hidden_size, input_size)   # output projection

    def forward(self, x: torch.Tensor, h: torch.Tensor = None):
        # x: [batch, seq_len, input_size]
        batch_size, seq_len, _ = x.shape
        if h is None:
            h = torch.zeros(batch_size, self.hidden_size)

        outputs = []
        for t in range(seq_len):
            # Core RNN equation: h_t = tanh(W_xh·x_t + W_hh·h_{t-1})
            h = torch.tanh(self.W_xh(x[:, t, :]) + self.W_hh(h))
            outputs.append(h)

        outputs = torch.stack(outputs, dim=1)  # [batch, seq_len, hidden]
        logits = self.W_hy(outputs)            # predict next token di setiap posisi
        return logits, h  # return final hidden state untuk carry-over

# Dengan nn.RNN (PyTorch built-in, lebih efisien)
class RNNLanguageModel(nn.Module):
    def __init__(self, vocab_size, embed_dim=256, hidden_size=512, num_layers=2):
        super().__init__()
        self.embedding = nn.Embedding(vocab_size, embed_dim)
        self.rnn = nn.RNN(
            input_size=embed_dim,
            hidden_size=hidden_size,
            num_layers=num_layers,
            dropout=0.3,          # dropout antar layer RNN
            batch_first=True,
        )
        self.fc = nn.Linear(hidden_size, vocab_size)

    def forward(self, token_ids, hidden=None):
        embedded = self.embedding(token_ids)     # [batch, seq, embed]
        output, hidden = self.rnn(embedded, hidden)
        logits = self.fc(output)                  # [batch, seq, vocab_size]
        return logits, hidden

2.4 LSTM & GRU // gating sebagai solusi

Long Short-Term Memory (LSTM) dan Gated Recurrent Unit (GRU) lahir dari satu insight fundamental: masalah vanishing gradient bisa diatasi jika kita punya jalur gradien langsung (gradient highway) yang tidak terpengaruh perkalian berulang. Mekanisme gating adalah cara menciptakan jalur tersebut secara selektif: sebuah "pintu" yang bisa dibuka atau ditutup oleh jaringan sendiri berdasarkan konteks.

Cell state LSTM adalah kunci arsitektur ini. Berbeda dari hidden state biasa, cell state berubah secara linear (bukan melalui fungsi non-linear saturating seperti tanh) — memungkinkan gradien mengalir lebih lancar sepanjang waktu. Gate-gate yang ada (input, forget, output) mengontrol informasi apa yang masuk, apa yang "dilupakan", dan apa yang menjadi output.

Anatomy LSTM — empat komponen

  • Forget Gate (f_t = σ(W_f·[h_{t-1}, x_t])) — memutuskan berapa banyak cell state lama yang dipertahankan. Nilai mendekati 0 = "lupakan", mendekati 1 = "ingat".
  • Input Gate (i_t = σ(W_i·[h_{t-1}, x_t])) — memutuskan nilai baru mana yang ditambahkan ke cell state.
  • Cell Update (C̃_t = tanh(W_C·[h_{t-1}, x_t])) — kandidat nilai baru yang akan ditambahkan.
  • Output Gate (o_t = σ(W_o·[h_{t-1}, x_t])) — memutuskan bagian cell state mana yang menjadi hidden output.
Pythonlstm_gru_comparison.py
import torch
import torch.nn as nn

class LSTMModel(nn.Module):
    def __init__(self, vocab_size, embed_dim=256, hidden_size=512, num_layers=2):
        super().__init__()
        self.embedding = nn.Embedding(vocab_size, embed_dim)
        self.lstm = nn.LSTM(
            input_size=embed_dim,
            hidden_size=hidden_size,
            num_layers=num_layers,
            dropout=0.3,
            batch_first=True,
            bidirectional=False,  # True untuk classification (bukan generation)
        )
        self.fc = nn.Linear(hidden_size, vocab_size)

    def forward(self, x, hidden=None):
        embedded = self.embedding(x)
        # LSTM returns (output, (h_n, c_n)) — beda dari RNN yang hanya (output, h_n)
        output, (h_n, c_n) = self.lstm(embedded, hidden)
        return self.fc(output), (h_n, c_n)

class GRUModel(nn.Module):
    """GRU: arsitektur lebih sederhana dari LSTM, performa seringkali setara."""
    def __init__(self, vocab_size, embed_dim=256, hidden_size=512, num_layers=2):
        super().__init__()
        self.embedding = nn.Embedding(vocab_size, embed_dim)
        self.gru = nn.GRU(embed_dim, hidden_size, num_layers, dropout=0.3, batch_first=True)
        self.fc = nn.Linear(hidden_size, vocab_size)

    def forward(self, x, hidden=None):
        embedded = self.embedding(x)
        output, h_n = self.gru(embedded, hidden)  # GRU tidak punya cell state
        return self.fc(output), h_n

# GRU vs LSTM comparison:
# GRU: 2 gates (reset, update) vs LSTM: 3 gates + cell state
# GRU: lebih sedikit parameter → lebih cepat dilatih
# GRU: performa seringkali match LSTM untuk sequence pendek-sedang
# LSTM: lebih baik untuk sequence sangat panjang karena cell state

# Contoh: bidirectional LSTM untuk classification
class BiLSTMClassifier(nn.Module):
    def __init__(self, vocab_size, embed_dim=128, hidden=256, num_classes=3):
        super().__init__()
        self.emb = nn.Embedding(vocab_size, embed_dim)
        self.lstm = nn.LSTM(embed_dim, hidden, bidirectional=True, batch_first=True)
        self.fc = nn.Linear(hidden * 2, num_classes)  # *2 karena bidirectional

    def forward(self, x):
        embedded = self.emb(x)
        output, (h_n, _) = self.lstm(embedded)
        # Concatenate forward & backward final hidden states
        h = torch.cat([h_n[-2], h_n[-1]], dim=1)
        return self.fc(h)
LSTM vs GRU — Perbandingan Arsitektur
LSTM — lebih powerful
Parameter per unit: 4 × (input + hidden + 1)
Gate: forget, input, output + cell state
Memory: explicit cell state (long-term)
Sequence: sangat panjang (1000+ token)
Parameter count: lebih banyak → lebih lambat
GRU — lebih efisien
Parameter per unit: 3 × (input + hidden + 1)
Gate: reset, update (tidak ada cell state)
Memory: implisit dalam hidden state
Sequence: pendek-sedang (<500 token)
Parameter count: lebih sedikit → lebih cepat
Kapan masih pakai LSTM/GRU di 2024?Untuk sebagian besar NLP tasks skala besar, Transformer adalah pilihan yang jelas. Tapi LSTM/GRU masih relevan untuk: deployment di embedded systems dengan memory sangat terbatas, time series prediction dengan sequence sangat panjang (10K+ timestep dimana Transformer quadratic attention menjadi bottleneck), dan online learning dimana model harus memproses data streaming satu token per waktu.

2.5 Seq2Seq & Encoder-Decoder // jembatan dua urutan

Arsitektur Seq2Seq (Sequence-to-Sequence) adalah breakthrough yang memungkinkan machine translation neural end-to-end. Ide dasarnya elegan: satu RNN (encoder) membaca input sequence dan mengkompresnya menjadi sebuah vektor tetap (context vector), lalu RNN kedua (decoder) menghasilkan output sequence dari context vector tersebut.

Namun, context vector tunggal adalah bottleneck fundamental: seluruh informasi dari input sequence sepanjang apapun harus dikompres menjadi satu vektor berdimensi fixed. Keterbatasan inilah yang memotivasi lahirnya mekanisme attention — alih-alih memaksa encoder meringkas semuanya ke satu vektor, biarkan decoder "melihat" langsung ke seluruh output encoder dan memilih mana yang relevan di setiap langkah decoding.

Pythonseq2seq.py
import torch
import torch.nn as nn

class Encoder(nn.Module):
    def __init__(self, src_vocab, embed_dim, hidden_size, num_layers=2):
        super().__init__()
        self.embedding = nn.Embedding(src_vocab, embed_dim)
        self.lstm = nn.LSTM(embed_dim, hidden_size, num_layers, batch_first=True)

    def forward(self, src):
        embedded = self.embedding(src)
        outputs, (h_n, c_n) = self.lstm(embedded)
        # outputs: [batch, src_len, hidden] — semua hidden states
        # h_n, c_n: final state — "summary" dari seluruh input
        return outputs, h_n, c_n

class Decoder(nn.Module):
    def __init__(self, tgt_vocab, embed_dim, hidden_size, num_layers=2):
        super().__init__()
        self.embedding = nn.Embedding(tgt_vocab, embed_dim)
        self.lstm = nn.LSTM(embed_dim, hidden_size, num_layers, batch_first=True)
        self.fc = nn.Linear(hidden_size, tgt_vocab)

    def forward(self, tgt_token, h, c):
        # tgt_token: [batch, 1] — satu token per langkah
        embedded = self.embedding(tgt_token)
        output, (h, c) = self.lstm(embedded, (h, c))
        prediction = self.fc(output.squeeze(1))  # [batch, tgt_vocab]
        return prediction, h, c

class Seq2Seq(nn.Module):
    def __init__(self, encoder, decoder):
        super().__init__()
        self.encoder = encoder
        self.decoder = decoder

    def forward(self, src, tgt, teacher_forcing_ratio=0.5):
        # teacher_forcing: kadang beri ground truth sebagai input decoder
        # (bukan output prediksi) untuk training lebih stabil
        enc_outputs, h, c = self.encoder(src)

        tgt_len = tgt.shape[1]
        predictions = []
        input_token = tgt[:, :1]  # BOS token

        for t in range(1, tgt_len):
            pred, h, c = self.decoder(input_token, h, c)
            predictions.append(pred)
            # Teacher forcing: pakai ground truth atau prediksi sebelumnya?
            use_gt = torch.rand(1).item() < teacher_forcing_ratio
            input_token = tgt[:, t:t+1] if use_gt else pred.argmax(1, keepdim=True)

        return torch.stack(predictions, dim=1)  # [batch, tgt_len-1, vocab]

2.6 Regularisasi & Training Tips // agar model generalisasi

Model neural network yang besar hampir selalu overfit pada training data jika dilatih tanpa kontrol. Regularisasi adalah kumpulan teknik untuk memaksa model belajar pola yang benar-benar general — bukan memorize training data. Untuk NLP tasks, regularisasi yang tepat seringkali perbedaan antara model yang bagus di training tapi jelek di production vs model yang robust.

Learning rate schedule adalah aspek training yang underrated tapi sangat krusial. Model yang dilatih dengan learning rate tetap hampir tidak pernah optimal — terlalu besar di akhir training (overshooting) atau terlalu kecil di awal (lambat konvergen). Warmup + decay adalah standar de facto untuk semua model NLP modern.

import torch
import torch.nn as nn
from torch.optim.lr_scheduler import LambdaLR
import math

# === Dropout — teknik regularisasi paling efektif untuk NLP ===
class RobustLSTM(nn.Module):
    def __init__(self, vocab_size, embed_dim=256, hidden=512):
        super().__init__()
        self.emb = nn.Embedding(vocab_size, embed_dim)
        self.emb_dropout = nn.Dropout(0.2)    # dropout setelah embedding
        self.lstm = nn.LSTM(embed_dim, hidden, num_layers=2,
                             dropout=0.3,     # dropout antar LSTM layers
                             batch_first=True)
        self.out_dropout = nn.Dropout(0.3)   # dropout sebelum output layer
        self.fc = nn.Linear(hidden, vocab_size)

    def forward(self, x, hidden=None):
        embedded = self.emb_dropout(self.emb(x))
        output, hidden = self.lstm(embedded, hidden)
        output = self.out_dropout(output)
        return self.fc(output), hidden

# === Warmup + Cosine Decay LR Schedule ===
def get_cosine_schedule_with_warmup(optimizer, warmup_steps: int, total_steps: int):
    def lr_lambda(current_step: int):
        if current_step < warmup_steps:
            return current_step / max(1, warmup_steps)  # linear warmup
        progress = (current_step - warmup_steps) / max(1, total_steps - warmup_steps)
        return max(0.0, 0.5 * (1.0 + math.cos(math.pi * progress)))  # cosine decay
    return LambdaLR(optimizer, lr_lambda)

# === Gradient Clipping — mencegah exploding gradients ===
def train_step(model, batch, optimizer, scheduler, max_grad_norm=1.0):
    model.train()
    optimizer.zero_grad(set_to_none=True)  # lebih efisien dari zero_grad()
    loss = compute_loss(model, batch)
    loss.backward()

    # Clip gradients — critical untuk LSTM/RNN training
    grad_norm = nn.utils.clip_grad_norm_(model.parameters(), max_grad_norm)

    optimizer.step()
    scheduler.step()

    return loss.item(), grad_norm.item()  # monitor grad_norm — jika > 10 ada masalah

# === Early Stopping — hentikan training saat val loss tidak membaik ===
class EarlyStopping:
    def __init__(self, patience: int = 5, min_delta: float = 1e-4):
        self.patience = patience
        self.counter = 0
        self.best_loss = float("inf")

    def __call__(self, val_loss: float) -> bool:
        if val_loss < self.best_loss - self.min_delta:
            self.best_loss = val_loss
            self.counter = 0
        else:
            self.counter += 1
        return self.counter >= self.patience  # True = stop training
Layer Norm vs Batch Norm untuk NLPBatch Normalization tidak bekerja baik untuk NLP karena panjang sequence bervariasi dan batch size sering kecil. Layer Normalization (nn.LayerNorm) menormalisasi di atas feature dimension per sample — tidak bergantung pada batch size dan bekerja baik untuk sequence variable length. Semua Transformer modern menggunakan Layer Norm, bukan Batch Norm.
