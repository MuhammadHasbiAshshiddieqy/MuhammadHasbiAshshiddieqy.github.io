3.1 Object Detection Fundamentals // menemukan dan melokalisasi objek

Object detection menjawab dua pertanyaan sekaligus: apa yang ada di gambar, dan di mana. Output: kumpulan bounding box (x, y, width, height) + class label + confidence score. Jauh lebih kompleks dari classification karena jumlah objek dan lokasinya tidak diketahui sebelumnya.

Dua paradigma utama

Two-stage (R-CNN family): tahap 1 propose region candidates, tahap 2 classify dan refine box. Akurat tapi lambat. Faster R-CNN adalah standar dua tahap terbaik.

One-stage (YOLO, SSD, RetinaNet): langsung predict box dan class dari grid gambar dalam satu pass. Lebih cepat, cocok untuk real-time. YOLO adalah raja one-stage.

Konsep penting: anchors dan IoU

Anchor boxes: prior shapes yang mewakili ukuran objek umum dalam dataset. Model memprediksi offset dari anchor, bukan koordinat absolut — membuat training lebih stabil. YOLO modern (v8+) sudah anchor-free.

NMS (Non-Maximum Suppression): ketika banyak box mendeteksi objek yang sama, simpan hanya yang tertinggi confidence-nya dan hapus yang overlap (IoU > threshold).

Pythondetection_basics.py
import torch
import torchvision

# Hitung IoU antara dua bounding box
def box_iou(box1, box2):
    """box format: [x1, y1, x2, y2]"""
    x1 = max(box1[0], box2[0])
    y1 = max(box1[1], box2[1])
    x2 = min(box1[2], box2[2])
    y2 = min(box1[3], box2[3])
    intersection = max(0, x2-x1) * max(0, y2-y1)
    area1 = (box1[2]-box1[0]) * (box1[3]-box1[1])
    area2 = (box2[2]-box2[0]) * (box2[3]-box2[1])
    union = area1 + area2 - intersection
    return intersection / union if union > 0 else 0

# Non-Maximum Suppression manual
def nms(boxes, scores, iou_thresh=0.5):
    indices = scores.argsort(descending=True)
    keep = []
    while indices.numel() > 0:
        i = indices[0].item()
        keep.append(i)
        if indices.numel() == 1: break
        ious = torch.tensor([box_iou(boxes[i], boxes[j]) for j in indices[1:]])
        indices = indices[1:][ious < iou_thresh]
    return keep

# torchvision built-in NMS
keep = torchvision.ops.nms(boxes=boxes_tensor, scores=scores_tensor, iou_threshold=0.5)

3.2 YOLO Family // you only look once — standar industri real-time

YOLO (You Only Look Once) adalah keluarga model detection yang mendominasi industri karena trade-off speed/accuracy yang unggul. YOLOv8 dan YOLO11 (2024) dari Ultralytics adalah versi paling modern — satu API untuk detection, segmentation, pose estimation, dan classification. Ultralytics Python package membuat training dan inference sangat mudah.

Evolusi YOLO

  • v1-v3: fondasi arsitektur grid-based one-stage detection
  • v5 (Ultralytics): popularisasi ekosistem Python yang mudah, anchor-based
  • v8 (2023): anchor-free, unified API untuk semua task, state-of-the-art
  • v9: PGI dan GELAN untuk efisiensi lebih tinggi
  • v11 / YOLO11 (2024): arsitektur baru, parameter lebih sedikit, akurasi lebih tinggi
  • RT-DETR: Transformer-based real-time detector dari Baidu, bersaing dengan YOLO

Custom training YOLO

Untuk custom dataset: (1) siapkan gambar + label YOLO format (x_center y_center width height class, normalized 0-1), (2) buat data.yaml dengan path dan nama kelas, (3) train dengan model.train(). Ultralytics mengurus augmentasi, learning rate schedule, dan evaluasi otomatis. Untuk 100-1000 gambar: fine-tune dari pretrained. Untuk <100 gambar: coba few-shot atau synthetic data.

Pythonyolo_detection.py
from ultralytics import YOLO

# Inference dengan YOLOv8 pretrained (80 kelas COCO)
model = YOLO("yolov8n.pt")   # n=nano, s=small, m=medium, l=large, x=xlarge
results = model.predict("foto.jpg", conf=0.5, iou=0.45)

for r in results:
    boxes = r.boxes.xyxy   # [N, 4] tensor: x1,y1,x2,y2
    confs = r.boxes.conf   # [N] confidence scores
    clss  = r.boxes.cls    # [N] class indices
    for box, conf, cls in zip(boxes, confs, clss):
        print(f"{model.names[int(cls)]}: {conf:.2f} @ {box.tolist()}")

# Custom training
model = YOLO("yolov8n.pt")   # mulai dari pretrained
results = model.train(
    data="data.yaml",         # path ke config dataset
    epochs=100,
    imgsz=640,
    batch=16,
    device="cuda",
    patience=20,              # early stopping
    augment=True,
    project="runs/train",
    name="custom_detector"
)

# Evaluasi pada validation set
metrics = model.val()
print(f"mAP50: {metrics.box.map50:.3f}")
print(f"mAP50-95: {metrics.box.map:.3f}")

# Export untuk deployment
model.export(format="onnx", imgsz=640, dynamic=True)
model.export(format="engine")   # TensorRT
3.3 Semantic Segmentation // klasifikasi di level piksel

Semantic segmentation mengklasifikasikan setiap piksel gambar ke kelas tertentu. Berbeda dengan detection yang hanya bounding box, segmentasi memberikan boundary yang tepat. Aplikasi: self-driving (aspal, trotoar, pejalan kaki), medical imaging (tumor boundary), satellite imagery (land use).

Arsitektur segmentasi

  • FCN: fully convolutional, pertama yang melakukan end-to-end pixel classification
  • U-Net: encoder-decoder dengan skip connections — dominan di medical imaging
  • DeepLab v3+: atrous convolution + ASPP untuk multi-scale context
  • SegFormer: Transformer-based, efficient, state-of-the-art per 2024
  • Mask2Former: unified architecture untuk semantic, instance, dan panoptic segmentation

U-Net: kenapa dominan di medical

Skip connections dari encoder ke decoder mempertahankan detail spasial yang hilang saat downsampling. Didesain untuk data medis yang terbatas — bekerja baik dengan ratusan gambar. Arsitektur simetris memudahkan implementasi. Decoding path mengupscale feature map dan concatenate dengan encoder features untuk prediksi piksel yang akurat.

Pythonsemantic_segmentation.py
import torch
import torch.nn as nn

# U-Net sederhana
class UNet(nn.Module):
    def __init__(self, in_ch=3, out_ch=1):
        super().__init__()
        def block(ic, oc):
            return nn.Sequential(
                nn.Conv2d(ic, oc, 3, padding=1), nn.BatchNorm2d(oc), nn.ReLU(True),
                nn.Conv2d(oc, oc, 3, padding=1), nn.BatchNorm2d(oc), nn.ReLU(True),
            )
        self.enc1 = block(in_ch, 64);  self.pool1 = nn.MaxPool2d(2)
        self.enc2 = block(64, 128);   self.pool2 = nn.MaxPool2d(2)
        self.bottleneck = block(128, 256)
        self.up2 = nn.ConvTranspose2d(256, 128, 2, stride=2)
        self.dec2 = block(256, 128)   # 128 (up) + 128 (skip) = 256
        self.up1 = nn.ConvTranspose2d(128, 64, 2, stride=2)
        self.dec1 = block(128, 64)    # 64 (up) + 64 (skip) = 128
        self.out  = nn.Conv2d(64, out_ch, 1)

    def forward(self, x):
        e1 = self.enc1(x)
        e2 = self.enc2(self.pool1(e1))
        b  = self.bottleneck(self.pool2(e2))
        d2 = self.dec2(torch.cat([self.up2(b), e2], dim=1))
        d1 = self.dec1(torch.cat([self.up1(d2), e1], dim=1))
        return self.out(d1)   # shape: (B, out_ch, H, W)

3.4 Instance Segmentation // setiap objek punya mask sendiri

Instance segmentation menggabungkan detection dan segmentation: setiap instance objek terdeteksi dengan bounding box DAN mendapat mask piksel individualnya. Dua anjing di gambar = dua mask terpisah. Berbeda dengan semantic segmentation yang semua anjing jadi satu kelas tanpa pemisahan instance.

Mask R-CNN

Extend Faster R-CNN dengan branch mask. Untuk setiap proposal region: (1) detect box dan classify seperti biasa, (2) tambahan: predict binary mask (28×28) untuk setiap kelas yang terdeteksi. RoI Align (bukan RoI Pool) untuk alignment piksel yang akurat. Lambat tapi sangat akurat.

YOLO segmentation dan SAM

YOLOv8-seg / YOLO11-seg: tambahkan mask prediction head ke YOLO detection. Real-time, mudah pakai, cukup akurat untuk production. SAM (Segment Anything Model): model fondasi dari Meta — bisa segmentasi objek APA PUN dari prompt (titik, box, atau teks). Tidak butuh training, tapi lebih lambat dari YOLO-seg.

Pythoninstance_segmentation.py
from ultralytics import YOLO
import numpy as np

# YOLO instance segmentation
model = YOLO("yolov8n-seg.pt")
results = model.predict("foto.jpg")

for r in results:
    if r.masks is not None:
        masks = r.masks.data    # [N, H, W] binary masks
        boxes = r.boxes.xyxy   # [N, 4]
        for i, (mask, box) in enumerate(zip(masks, boxes)):
            mask_np = mask.cpu().numpy().astype(np.uint8) * 255
            print(f"Object {i}: mask sum={mask_np.sum()}, box={box.tolist()}")

# SAM — Segment Anything (prompt-based)
from segment_anything import SamPredictor, sam_model_registry
import cv2

sam = sam_model_registry["vit_b"](checkpoint="sam_vit_b.pth")
predictor = SamPredictor(sam)

img = cv2.imread("foto.jpg")
predictor.set_image(cv2.cvtColor(img, cv2.COLOR_BGR2RGB))

# Segmentasi dari titik prompt (x, y)
masks, scores, _ = predictor.predict(
    point_coords=np.array([[300, 200]]),
    point_labels=np.array([1]),   # 1=foreground, 0=background
    multimask_output=True
)
best_mask = masks[scores.argmax()]   # pilih mask terbaik
3.5 Keypoint Detection & Pose Estimation // memahami struktur tubuh dan objek

Pose estimation mendeteksi keypoint — titik-titik penting pada tubuh (sendi: bahu, siku, lutut) atau objek. Aplikasi: motion capture, fitness tracking, gesture recognition, robot manipulation, analisis biomechanical.

Top-down vs bottom-up

Top-down: detect orang dulu (YOLO/detection), kemudian predict keypoint dalam setiap bounding box. Akurat, tapi lambat ketika banyak orang (N orang = N inference). HRNet, ViTPose.

Bottom-up: detect semua keypoint sekaligus, kemudian asosiasikan ke individu. Lebih cepat untuk banyak orang, tapi asosiasi lebih sulit. OpenPose, HigherHRNet.

Output format dan post-processing

Model biasanya menghasilkan heatmap — satu heatmap per keypoint, menunjukkan probabilitas lokasi. Argmax heatmap = koordinat keypoint. Untuk 17-keypoint COCO format: hidung, mata, telinga, bahu, siku, pergelangan tangan, pinggul, lutut, pergelangan kaki. Skeleton digambar dengan menghubungkan pasangan keypoint sesuai definisi.

Pythonpose_estimation.py
from ultralytics import YOLO
import cv2
import numpy as np

# YOLO pose estimation (17 keypoints COCO)
model = YOLO("yolov8n-pose.pt")
results = model.predict("video.mp4", stream=True)

# COCO skeleton: pasangan keypoint yang dihubungkan
SKELETON = [
    [15,13],[13,11],[16,14],[14,12],  # kaki
    [11,12],[5,11],[6,12],              # pinggul-bahu
    [5,6],[5,7],[6,8],[7,9],[8,10],   # lengan
    [1,2],[0,1],[0,2],[1,3],[2,4],     # kepala
    [3,5],[4,6]                          # leher
]

for r in results:
    frame = r.orig_img.copy()
    if r.keypoints is not None:
        kps = r.keypoints.xy.cpu().numpy()  # [N_persons, 17, 2]
        kp_conf = r.keypoints.conf.cpu().numpy()  # [N_persons, 17]
        for person_kps, person_conf in zip(kps, kp_conf):
            # Gambar keypoints
            for (x, y), conf in zip(person_kps, person_conf):
                if conf > 0.5:
                    cv2.circle(frame, (int(x),int(y)), 4, (0,255,0), -1)
            # Gambar skeleton
            for (i, j) in SKELETON:
                if person_conf[i] > 0.5 and person_conf[j] > 0.5:
                    p1 = (int(person_kps[i,0]),int(person_kps[i,1]))
                    p2 = (int(person_kps[j,0]),int(person_kps[j,1]))
                    cv2.line(frame, p1, p2, (255,128,0), 2)
