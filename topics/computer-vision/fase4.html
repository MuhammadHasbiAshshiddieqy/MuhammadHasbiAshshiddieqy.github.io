4A.1 Model Optimization // lebih cepat, lebih ringan, tanpa kehilangan akurasi

Di production, model yang bagus tapi lambat tidak berguna. Optimasi model vision berfokus pada tiga dimensi: latency (waktu inference satu input), throughput (berapa request per detik), dan memory footprint. Teknik utama: pruning, knowledge distillation, dan architectural efficiency.

Pruning dan sparsity

Structured pruning: hapus seluruh filter/channel yang kontribusinya kecil (diukur dengan magnitude atau gradient). Menghasilkan model lebih kecil yang tetap bisa jalan efisien di hardware biasa.

Unstructured pruning: set bobot individual ke nol. Compression ratio tinggi tapi butuh sparse inference support (hardware-specific).

Setelah pruning: fine-tune beberapa epoch untuk recovery akurasi.

Knowledge Distillation

Train student model kecil untuk meniru teacher model besar. Student belajar dari "soft labels" (distribusi probabilitas teacher) bukan hard ground truth — membawa informasi lebih kaya tentang similarity antar kelas. Efektif untuk compress model 4–10× dengan kehilangan akurasi minimal. Teknik: response-based (mimic logits), feature-based (mimic intermediate features), relation-based (mimic feature relationships).

Pythonmodel_optimization.py
import torch
import torch.nn as nn
import torch.nn.utils.prune as prune

# Structured pruning — hapus 20% channel terkecil di Conv layers
def apply_pruning(model, amount=0.2):
    for name, module in model.named_modules():
        if isinstance(module, nn.Conv2d):
            prune.ln_structured(module, name='weight', amount=amount, n=2, dim=0)
    return model

# Knowledge Distillation loss
def distillation_loss(student_logits, teacher_logits, labels, T=4, alpha=0.7):
    soft_loss = nn.functional.kl_div(
        nn.functional.log_softmax(student_logits / T, dim=-1),
        nn.functional.softmax(teacher_logits / T, dim=-1),
        reduction='batchmean'
    ) * (T**2)
    hard_loss = nn.functional.cross_entropy(student_logits, labels)
    return alpha * soft_loss + (1 - alpha) * hard_loss

# Benchmark latency
import time
def benchmark(model, input_size=(1,3,224,224), n=100):
    x = torch.randn(*input_size).cuda()
    model = model.cuda().eval()
    # Warmup
    for _ in range(10): model(x)
    torch.cuda.synchronize()
    t0 = time.time()
    for _ in range(n): model(x)
    torch.cuda.synchronize()
    ms_per_image = (time.time() - t0) / n * 1000
    print(f"Latency: {ms_per_image:.2f} ms/img ({1000/ms_per_image:.1f} FPS)")
4A.2 Quantization & TensorRT // float32 → int8 tanpa kehilangan kualitas

Quantization mengubah bobot model dari FP32 (4 byte per nilai) ke INT8 atau FP16 (1–2 byte). Hasil: ukuran model 4× lebih kecil, inference 2–4× lebih cepat, dan penggunaan memori GPU jauh berkurang. TensorRT dari NVIDIA mengoptimalkan model spesifik untuk hardware target dan bisa menggabungkan beberapa layer menjadi satu operasi (layer fusion).

Post-training quantization (PTQ)

Quantize model yang sudah di-train, tanpa training ulang. Butuh sedikit "calibration data" (~100–1000 sampel) untuk mengukur distribusi aktivasi per layer. Dua mode: static (calibrate aktivasi di inference time) vs dynamic (aktivasi dihitung saat runtime, bobot tetap INT8). Static lebih cepat untuk image models.

Quantization-aware training (QAT)

Simulasikan quantization selama training — model belajar untuk toleran terhadap precision loss. Hasil akurasi lebih baik dari PTQ, terutama untuk model kecil atau dataset sulit. Butuh training ulang 10–20% epochs dari full training. Digunakan ketika PTQ menghasilkan drop akurasi yang tidak dapat diterima.

4A.3 ONNX — Open Neural Network Exchange // portabilitas lintas framework

ONNX adalah format model terbuka yang memungkinkan model di-train di PyTorch/TensorFlow kemudian di-deploy di berbagai runtime (ONNX Runtime, TensorRT, OpenVINO, CoreML). Solusi untuk menghindari vendor lock-in dan untuk deploy ke platform yang tidak support PyTorch natively.

Export ke ONNX

PyTorch torch.onnx.export() melakukan trace forward pass model dengan dummy input dan merekam semua operasi sebagai graph ONNX. Penting: export dengan dynamic_axes untuk batch size dan dimensi spatial yang fleksibel. Selalu verifikasi output model ONNX sama dengan PyTorch original.

ONNX Runtime inference

ONNX Runtime (ORT) adalah inference engine cross-platform yang sangat dioptimalkan. Mendukung CPU, CUDA, TensorRT, OpenVINO, CoreML sebagai "execution providers". Latency biasanya lebih rendah dari PyTorch eager mode karena graph optimization dan operator fusion. Digunakan oleh production besar (Azure, Windows, Office).

Pythononnx_export_inference.py
import torch
import onnx
import onnxruntime as ort
import numpy as np

# Export PyTorch model ke ONNX
model.eval()
dummy = torch.randn(1, 3, 224, 224)
torch.onnx.export(
    model, dummy, "model.onnx",
    input_names=['image'],
    output_names=['logits'],
    dynamic_axes={'image': {0: 'batch', 2: 'height', 3: 'width'}},
    opset_version=17,
    do_constant_folding=True
)

# Verifikasi graph valid
onnx_model = onnx.load("model.onnx")
onnx.checker.check_model(onnx_model)
print("ONNX model valid ✓")

# Verifikasi output sama dengan PyTorch
with torch.no_grad(): torch_out = model(dummy).numpy()
ort_session = ort.InferenceSession("model.onnx")
ort_out = ort_session.run(None, {'image': dummy.numpy()})[0]
np.testing.assert_allclose(torch_out, ort_out, rtol=1e-3, atol=1e-5)

# Inference dengan ONNX Runtime (CUDA provider)

    "model.onnx", sess_opts,
    providers=['CUDAExecutionProvider', 'CPUExecutionProvider']
)
inputs = {'image': preprocessed_img.numpy()}

4A.4 Edge Deployment — Jetson & Raspberry Pi // vision di luar cloud

Edge deployment berarti menjalankan model CV langsung di perangkat — bukan di cloud. Keuntungan: latency rendah (tidak perlu network round-trip), privasi data (tidak keluar ke server), operasi offline, dan biaya lebih rendah untuk scale. Dua platform paling populer: NVIDIA Jetson (untuk akurasi tinggi) dan Raspberry Pi (untuk biaya rendah).

NVIDIA Jetson Orin

GPU embedded dengan dukungan CUDA, TensorRT, dan DeepStream penuh. Jetson Nano ~$100 untuk project kecil, Jetson Orin Nano–NX untuk production serius. Workflow: train di workstation → export TensorRT engine → copy ke Jetson → inference dengan ONNX Runtime atau TensorRT direct. Power consumption 5–15W vs GPU server 200–400W.

Raspberry Pi + Hailo

RPi 5 dengan Hailo-8 M.2 AI Kit memberikan 26 TOPS untuk ~$70. Model perlu diconvert ke format Hailo (HEF) menggunakan Dataflow Compiler. Alternatif: gunakan RPi dengan model sangat kecil (MobileNet, EfficientNet-lite) dan quantization INT8. Kecepatan terbatas tapi cukup untuk deteksi objek sederhana pada 5–15 FPS.

Pythonjetson_inference.py
# Jalankan di Jetson dengan TensorRT engine
import tensorrt as trt
import pycuda.driver as cuda
import numpy as np

class TRTInference:
    def __init__(self, engine_path):
        logger = trt.Logger(trt.Logger.WARNING)
        with open(engine_path, 'rb') as f:
            self.engine = trt.Runtime(logger).deserialize_cuda_engine(f.read())
        self.context = self.engine.create_execution_context()
        self.stream = cuda.Stream()
        self.inputs, self.outputs, self.bindings = [], [], []
        for binding in self.engine:
            size = trt.volume(self.engine.get_tensor_shape(binding))
            host_mem   = cuda.pagelocked_empty(size, np.float32)
            device_mem = cuda.mem_alloc(host_mem.nbytes)
            self.bindings.append(int(device_mem))
            if self.engine.get_tensor_mode(binding) == trt.TensorIOMode.INPUT:
                self.inputs.append({'host': host_mem, 'device': device_mem})
            else:
                self.outputs.append({'host': host_mem, 'device': device_mem})

    def infer(self, image_np):
        np.copyto(self.inputs[0]['host'], image_np.ravel())
        cuda.memcpy_htod_async(self.inputs[0]['device'], self.inputs[0]['host'], self.stream)
        self.context.execute_async_v2(self.bindings, self.stream.handle)
        cuda.memcpy_dtoh_async(self.outputs[0]['host'], self.outputs[0]['device'], self.stream)
        self.stream.synchronize()
        return self.outputs[0]['host']
4A.5 Real-time Vision Pipeline // kamera → inference → aksi dalam milidetik

Pipeline real-time bukan hanya soal model yang cepat — ini adalah sistem end-to-end: capture kamera, preprocessing, batching, inference, post-processing, dan output ke downstream system. Bottleneck sering bukan di model, tapi di I/O, preprocessing, atau data transfer CPU↔GPU.

Pipeline profiling

Ukur setiap stage secara terpisah sebelum optimasi. Preprocessing sering bottleneck yang diabaikan: decoding JPEG, resize, normalize, to-tensor. Solusi: pindahkan ke GPU dengan torchvision transforms (CUDA), atau gunakan DALI (NVIDIA Data Loading Library) untuk full GPU preprocessing pipeline.

Asynchronous capture dan inference

Jangan block inference thread untuk menunggu frame baru. Pakai producer-consumer pattern: dedicated thread untuk capture frames ke queue, inference thread pull dari queue. Dengan GPU stream asynchrony, transfer H2D, inference, dan D2H bisa overlap. Target: CPU tidak pernah idle menunggu GPU.

Pythonrealtime_pipeline.py
import cv2
import threading
import queue
from ultralytics import YOLO

class RTVisionPipeline:
    def __init__(self, source, model_path, max_queue=4):
        self.cap   = cv2.VideoCapture(source)
        self.model = YOLO(model_path)
        self.frame_q  = queue.Queue(maxsize=max_queue)
        self.result_q = queue.Queue(maxsize=max_queue)
        self.running  = True

    def capture_thread(self):
        while self.running:
            ret, frame = self.cap.read()
            if not ret: break
            if not self.frame_q.full():
                self.frame_q.put_nowait(frame)  # drop frame if queue full (real-time)

    def inference_thread(self):
        while self.running:
            try:
                frame = self.frame_q.get(timeout=0.1)
                results = self.model.predict(frame, verbose=False)
                if not self.result_q.full():
                    self.result_q.put_nowait(results[0])
            except queue.Empty: continue

    def run(self):
        t_cap = threading.Thread(target=self.capture_thread, daemon=True)
        t_inf = threading.Thread(target=self.inference_thread, daemon=True)
        t_cap.start(); t_inf.start()
        while self.running:
            try:
                r = self.result_q.get(timeout=0.1)
                vis = r.plot()
                cv2.imshow('Vision', vis)
                if cv2.waitKey(1) == 27: self.running = False
            except queue.Empty: continue
        self.cap.release(); cv2.destroyAllWindows()

4B.1 CLIP & Vision-Language Models // teks dan gambar dalam satu ruang

CLIP (Contrastive Language-Image Pre-Training, OpenAI 2021) melatih image encoder dan text encoder secara bersamaan menggunakan contrastive learning — gambar dan caption yang cocok didekatkan, yang tidak dijauhkan. Hasilnya: embedding space yang bisa dibandingkan lintas modalitas. Ini adalah fondasi dari hampir semua VLM modern.

Zero-shot classification

Tanpa training, CLIP bisa classify gambar ke kelas APA PUN: encode gambar dan deskripsi kelas ("a photo of a cat", "a photo of a dog"), hitung cosine similarity, pilih kelas tertinggi. Fleksibel luar biasa — tidak butuh annotated data. Akurasi tidak sebaik fine-tuned model tapi cukup untuk prototype cepat.

VLM lebih modern

SigLIP (Google): CLIP dengan sigmoid loss, lebih efisien training
OpenCLIP: open-source reimplementasi dengan banyak varian
BLIP-2: tambahkan language generation — bisa menjawab pertanyaan tentang gambar
LLaVA: connect CLIP encoder ke LLM untuk visual instruction following
PaliGemma: VLM compact dari Google, cocok untuk fine-tuning

Pythonclip_vlm.py
import torch
from transformers import CLIPModel, CLIPProcessor
from PIL import Image

model = CLIPModel.from_pretrained("openai/clip-vit-base-patch32")

# Zero-shot classification

probs = outputs.logits_per_image.softmax(dim=1)[0]
for label, prob in zip(labels, probs):
    print(f"{label}: {prob.item():.3f}")

# Image embedding untuk similarity search

    return nn.functional.normalize(emb, dim=-1)

4B.2 Diffusion Model Fundamentals // belajar dari noise

Diffusion models adalah keluarga generative model yang mendominasi image generation sejak 2022. Ide utama: tambahkan noise Gaussian bertahap ke gambar (forward process) hingga jadi pure noise, kemudian train neural network untuk membalikkan proses ini (reverse process). Di inference time: mulai dari pure noise, ulangi denoising step, dapatkan gambar.

Komponen utama

  • U-Net denoiser: memprediksi noise yang harus dihilangkan. Menggunakan time embedding untuk tahu "seberapa banyak noise" saat ini
  • Noise scheduler: mengontrol jadwal penambahan/pengurangan noise (DDPM, DDIM, DPM-Solver). DDIM jauh lebih cepat dari DDPM
  • VAE: latent diffusion bekerja di compressed latent space, bukan pixel space — 4× lebih efisien
  • Text encoder: CLIP atau T5 mengkonvert text prompt ke kondisi untuk guiding denoising

Sampling dan guidance

Classifier-free guidance (CFG): teknik yang membuat model "lebih mendengar" prompt. CFG scale tinggi (7–12) = gambar lebih sesuai prompt tapi kurang diverse. Terlalu tinggi = artifacts.

Step count: 20–50 step biasanya cukup dengan DDIM/DPM-Solver. DDPM original butuh 1000 step. Trade-off: lebih banyak step = lebih detail, tapi lebih lambat.

4B.3 Stable Diffusion Fine-tuning // dreambooth, lora, dan textual inversion

Fine-tuning memungkinkan model SD belajar gaya visual, subjek spesifik, atau domain baru dari sedikit gambar (5–30 cukup). Tiga teknik utama: DreamBooth (strong concept binding), LoRA (parameter-efficient), dan Textual Inversion (embedding-only).

LoRA (Low-Rank Adaptation)

Tambahkan low-rank weight matrix ke layer attention U-Net, hanya train weight tambahan ini (~0.5–1% parameter asli). Hasil: file LoRA kecil (~50–200MB) yang bisa digunakan bersama model base apa pun. Bisa combine beberapa LoRA sekaligus. Paling populer karena efisien dan fleksibel.

DreamBooth

Fine-tune seluruh model SD (~2GB) pada 5–30 foto subjek + rare token identifier ("sks dog"). Butuh VRAM lebih besar dan training lebih lama. Hasil: konsistensi subjek yang sangat kuat — model "ingat" wajah/objek spesifik. Gunakan "prior preservation loss" untuk mencegah language drift pada kelas umum.

Pythonlora_finetuning.py
from diffusers import StableDiffusionPipeline

# Setup LoRA untuk U-Net attention layers

    lora_dropout=0.05
)

pipe = StableDiffusionPipeline.from_pretrained("runwayml/stable-diffusion-v1-5")
unet_with_lora = get_peft_model(pipe.unet, lora_config)
unet_with_lora.print_trainable_parameters()
# trainable params: ~800K || all params: 860M || trainable%: 0.09%

# Training loop
optimizer = torch.optim.AdamW(unet_with_lora.parameters(), lr=1e-4)
for step, batch in enumerate(train_loader):
    latents  = pipe.vae.encode(batch["pixel_values"]).latent_dist.sample() * 0.18215
    noise    = torch.randn_like(latents)
    timesteps = torch.randint(0, 1000, (latents.shape[0],))
    noisy_l  = pipe.scheduler.add_noise(latents, noise, timesteps)
    enc_out  = pipe.text_encoder(batch["input_ids"])[0]
    noise_pred = unet_with_lora(noisy_l, timesteps, encoder_hidden_states=enc_out).sample
    loss = nn.functional.mse_loss(noise_pred, noise)
    loss.backward(); optimizer.step(); optimizer.zero_grad()

# Save LoRA weights saja (~few MB)
unet_with_lora.save_pretrained("lora_weights")
4B.4 Image Editing & Inpainting // mengubah bagian gambar yang ada

Selain generate dari scratch, diffusion models bisa mengedit gambar yang ada. Inpainting: hapus objek atau isi area yang masked dengan konten baru. Img2img: ubah gaya atau konten gambar dengan mempertahankan struktur. InstructPix2Pix: edit gambar menggunakan instruksi bahasa alami.

Inpainting pipeline

Berikan gambar asli + mask (putih = area yang diedit, hitam = yang dipertahankan). Diffusion process hanya berjalan di area masked. Penting: inpainting model khusus (fine-tuned untuk task ini) jauh lebih baik dari model general purpose. SDXL Inpainting atau SD2 Inpainting adalah pilihan terbaik.

ControlNet

ControlNet menambahkan spatial conditioning ke diffusion — kontrol posisi, struktur, dan konten gambar. Input conditioning bisa: pose keypoints, depth map, canny edge, segmentation mask, atau sketsa. Satu model ControlNet bisa di-stack dengan ControlNet lain untuk multiple conditioning. Sangat powerful untuk generating consistent characters atau scenes.

4B.5 Video Generation // diffusion di dimensi temporal

Video generation adalah frontier terbaru generative CV. Model seperti Sora (OpenAI), CogVideoX, AnimateDiff, dan Wan2.1 memperluas diffusion ke dimensi waktu. Challenge utama: temporal consistency (objek tidak berubah antar frame), motion quality, dan computational cost yang jauh lebih tinggi dari image generation.

Pendekatan video generation

  • AnimateDiff: tambahkan temporal attention ke SD weights, generate video 16 frame
  • SVD (Stable Video Diffusion): image-to-video dari satu frame seed
  • CogVideoX: open-source, text-to-video 720p, 49 frames
  • Wan2.1 (2025): state-of-the-art open-source, bisa T2V dan I2V
  • Sora: closed, OpenAI, world model dengan physical consistency

Practical constraints

Video generation sangat VRAM-hungry. AnimateDiff 16 frame di 512px butuh ~12GB. CogVideoX 720p ~24GB. Teknik untuk mengurangi: tiled VAE decoding, CPU offloading, FP8 quantization. Di production: gunakan inference API (Replicate, Modal) daripada self-host kecuali ada A100/H100. Untuk preview/prototype: pakai platform seperti Kling, Hailuo, atau RunwayML API.
