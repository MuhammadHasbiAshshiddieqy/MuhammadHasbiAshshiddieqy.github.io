5.1 CV System Design // dari gambar input ke business outcome

Membangun sistem CV production bukan hanya tentang memilih model yang terbaik — ini tentang merancang sistem yang reliable, scalable, dan mudah di-maintain. Interview system design untuk CV engineer akan menanyakan pertanyaan seperti: "Desain sistem deteksi anomali untuk pabrik dengan 50 kamera." Kamu harus tahu trade-off di setiap layer.

Layer sistem CV

  1. Ingestion: kamera stream (RTSP), video upload, atau batch images. Pertimbangkan bandwidth, latency, storage.
  2. Preprocessing: resize, normalize, format conversion. Bisa di CPU atau GPU (DALI, torchvision CUDA).
  3. Inference: single model atau ensemble. Batch inference untuk throughput vs streaming untuk latency.
  4. Post-processing: NMS, filtering, tracking (DeepSORT), temporal smoothing.
  5. Storage: raw images, predictions, metadata. Object storage (S3/GCS) + time-series DB untuk metrics.
  6. Downstream: alerting, dashboard, API ke sistem lain, retraining pipeline.

Trade-off kritis

Latency vs throughput: batch size tinggi = throughput tinggi tapi latency lebih lama. Untuk real-time alert: batch=1. Untuk analytics pipeline: batch=32–64.

Accuracy vs speed: model besar lebih akurat tapi lebih lambat. Gunakan cascaded approach: model cepat dulu untuk filter, model besar hanya untuk confident candidates.

Cloud vs edge: edge untuk privasi dan low-latency, cloud untuk heavy processing dan storage. Hybrid: pre-filter di edge, kirim ke cloud hanya event penting.

Pythoncv_system_design.py
# Cascaded detection: fast model untuk filter, akurat untuk classify
class CascadeDetector:
    def __init__(self):
        from ultralytics import YOLO
        self.fast  = YOLO("yolov8n.engine")  # nano TRT — kecepatan
        self.accu  = YOLO("yolov8x.engine")  # xlarge TRT — akurasi
        self.conf_thresh = 0.3

    def predict(self, frame):
        # Stage 1: fast model — proses semua frame
        fast_results = self.fast.predict(frame, conf=self.conf_thresh, verbose=False)
        if not fast_results[0].boxes: return []
        # Stage 2: only if something detected — crop dan refine
        crops = [frame[int(b[1]):int(b[3]),int(b[0]):int(b[2])] for b in fast_results[0].boxes.xyxy]
        precise = [self.accu.predict(c, conf=0.5, verbose=False) for c in crops]
        return precise

# Multi-camera stream management
import asyncio

async def process_camera(cam_id, rtsp_url, detector, results_queue):
    import cv2
    cap = cv2.VideoCapture(rtsp_url)
    while True:
        ret, frame = cap.read()
        if not ret: break
        detections = detector.predict(frame)
        if detections:
            await results_queue.put({'cam': cam_id, 'detections': detections})
        await asyncio.sleep(0)  # yield to other coroutines
5.2 Serving Vision Models — Triton Inference Server // GPU utilization maksimal

NVIDIA Triton Inference Server adalah platform serving model yang dirancang khusus untuk AI workloads. Mendukung PyTorch, TensorRT, ONNX, TensorFlow, Python models, dan custom backends dalam satu server. Dynamic batching, model concurrency, dan pipeline orchestration out-of-the-box.

Fitur kunci Triton

  • Dynamic batching: auto-group incoming requests untuk maximize GPU utilization
  • Model concurrency: jalankan beberapa instance model secara paralel
  • Ensemble pipeline: chain beberapa model (preprocessor → detector → classifier) dalam satu request
  • Model repository: structure folder dengan config.pbtxt untuk setiap model
  • gRPC + HTTP: client library tersedia untuk Python, C++, Java

BentoML sebagai alternatif

Untuk proyek yang tidak butuh multi-GPU complexity, BentoML lebih mudah di-setup: dekorasi class dengan @bentoml.service, define endpoint dengan @bentoml.api, container-ready dengan bentoml containerize. Cocok untuk startup atau tim kecil. Triton untuk skala enterprise dengan SLA ketat dan traffic tinggi.

# Python client
import tritonclient.grpc as grpcclient
import numpy as np

client = grpcclient.InferenceServerClient(url="localhost:8001")

def triton_detect(image_np):
    # Preprocess: resize + normalize
    inp = grpcclient.InferInput("images", image_np.shape, "FP32")
    inp.set_data_from_numpy(image_np)
    output = grpcclient.InferRequestedOutput("output0")
    result = client.infer("yolov8n", inputs=[inp], outputs=[output])
    return result.as_numpy("output0")

5.3 Monitoring & Data Drift Detection // model yang bagus hari ini bisa jelek besok

Model CV production bisa degradasi tanpa terlihat. Kamera kotor, perubahan pencahayaan musiman, perubahan produk, atau distribusi traffic yang bergeser — semua bisa menyebabkan penurunan performa tanpa error apapun di sistem. Monitoring yang baik mendeteksi ini sebelum business impact terasa.

Apa yang dimonitor

  • Infrastructure: latency p50/p95/p99, throughput req/s, GPU utilization, memory usage, error rate
  • Model performance: confidence score distribution, class distribution prediction, bounding box size distribution
  • Data quality: image brightness, blur level, aspect ratio — perubahan signifikan = potensi pipeline issue
  • Business metrics: false positive rate di downstream (jika ada human review loop), precision@threshold

Drift detection strategies

Statistical tests: KS test atau chi-square untuk distribusi fitur antara training data dan production data. Alat: Evidently AI, Arize, WhyLabs.

Embedding drift: bandingkan distribution embedding visual antara window waktu berbeda menggunakan cosine similarity atau MMD. Paling robust untuk image data.

Shadow model: jalankan model baru paralel dengan production, bandingkan prediction — deteksi regresi sebelum deploy.

# Drift detector berbasis statistik
class ConfidenceDriftDetector:
    def __init__(self, reference_scores: list, alpha=0.01):
        self.reference = np.array(reference_scores)
        self.alpha = alpha
        self.window = []
        self.window_size = 500

    def update(self, score: float) -> dict:
        self.window.append(score)
        if len(self.window) > self.window_size:
            self.window.pop(0)
        if len(self.window) < 100: return {'drift': False}
        ks_stat, p_val = stats.ks_2samp(self.reference, np.array(self.window))
        return {'drift': p_val < self.alpha, 'p_value': p_val, 'ks_stat': ks_stat}

# Image quality check sebelum inference
def check_image_quality(img_np) -> dict:
    brightness = img_np.mean()
    blur_var   = cv2.Laplacian(cv2.cvtColor(img_np, cv2.COLOR_BGR2GRAY), cv2.CV_64F).var()
    IMAGE_BRIGHTNESS.set(brightness)
    return {
        'ok': brightness > 30 and blur_var > 100,
        'brightness': brightness,
        'sharpness': blur_var,
        'issues': [
            'too dark' if brightness < 30 else None,
            'too blurry' if blur_var < 100 else None
        ]
    }
5.4 Anotasi & Active Learning // data flywheel untuk model yang terus membaik

Model yang di-deploy menghadapi distribusi data baru yang terus berubah. Untuk mempertahankan performa, perlu retraining — tapi annotate semua data baru mahal. Active learning memilih sampel paling informatif untuk dianotasi: maximal improvement dengan minimal labeling cost.

Pipeline active learning untuk CV

  1. Collect: simpan sampel production yang menarik (confidence rendah, kelas langka, edge case)
  2. Score: hitung uncertainty score tiap sampel — confidence threshold, entropy, atau committee disagreement
  3. Sample: pilih N sampel tertinggi uncertainty + diversity sampling
  4. Annotate: tampilkan ke annotator, prioritaskan yang model paling tidak yakin
  5. Retrain: tambahkan anotasi baru ke training set, fine-tune model
  6. Deploy: eval dulu di holdout set sebelum ganti model production

Tools anotasi

  • Label Studio: open-source, mendukung image, video, audio, text. Self-hostable.
  • Roboflow: platform lengkap: anotasi + augmentasi + export ke berbagai format (YOLO, COCO, Pascal VOC)
  • CVAT: open-source, dari Intel, sangat powerful untuk detection dan segmentation
  • Scale AI / Labelbox: enterprise, managed annotation services dengan QA
  • SAM-assisted annotation: gunakan SAM untuk auto-mask suggestion — annotator hanya konfirmasi, bukan gambar manual
Pythonactive_learning.py
import numpy as np
from sklearn.cluster import KMeans

# Uncertainty sampling: pilih sampel dengan confidence paling rendah
def uncertainty_score(detections):
    """Aggregate uncertainty dari semua deteksi dalam satu gambar"""
    if not detections: return 1.0  # no detection = very uncertain
    confs = [d['confidence'] for d in detections]
    return 1.0 - np.mean(confs)

# Core-set sampling: maximize coverage via clustering
def coreset_sample(embeddings: np.ndarray, n_samples: int) -> list:
    """Pilih diverse subset dari kandidat menggunakan k-means clustering"""
    kmeans = KMeans(n_clusters=n_samples, random_state=42)
    kmeans.fit(embeddings)
    selected = []
    for center in kmeans.cluster_centers_:
        dists = np.linalg.norm(embeddings - center, axis=1)
        selected.append(int(dists.argmin()))
    return selected

# SAM-assisted annotation: auto-generate mask proposals
from segment_anything import SamPredictor, sam_model_registry
from ultralytics import YOLO

def auto_annotate(image, detector, sam_predictor):
    """Generate detection + mask proposals untuk human review"""
    det_results = detector.predict(image, conf=0.3, verbose=False)[0]
    sam_predictor.set_image(image)
    proposals = []
    for box, conf, cls in zip(det_results.boxes.xyxy, det_results.boxes.conf, det_results.boxes.cls):
        masks, scores, _ = sam_predictor.predict(box=box.cpu().numpy(), multimask_output=False)
        proposals.append({'box': box.tolist(), 'class': int(cls), 'confidence': float(conf), 'mask': masks[0]})
    return proposals  # kirim ke annotation platform untuk review
5.5 Checklist CV Engineer // semua yang harus kamu kuasai

Ini adalah checklist komprehensif kemampuan yang diharapkan dari seorang CV Engineer yang siap production. Review ini secara jujur — area mana yang masih perlu penguatan?

Image Processing & Fondasi

  • Representasi gambar sebagai NumPy array, manipulasi channel BGR/RGB/HSV
  • Implementasi filter konvolusi manual dengan numpy (tanpa library CV)
  • Canny edge detection, transformasi geometris, histogram equalization
  • Desain augmentasi pipeline dengan Albumentations untuk classification dan detection

Deep Learning CV

  • Implementasi CNN dengan BatchNorm, skip connections, dan AdaptiveAvgPool dari scratch
  • Fine-tune pretrained model (ResNet, EfficientNet, ViT) dengan layer-wise learning rate
  • Implementasi Focal Loss dan Dice Loss untuk imbalanced dataset
  • Evaluasi model: mAP untuk detection, mIoU untuk segmentation, confusion matrix

Detection & Segmentation

  • Training YOLOv8/v11 pada dataset custom, anotasi dengan LabelImg atau Roboflow
  • Implementasi dan debug pipeline NMS secara manual
  • Semantic segmentation dengan U-Net, evaluasi mIoU
  • Instance segmentation dengan YOLO-seg, penggunaan SAM untuk prompt-based segmentation
  • Pose estimation dengan keypoint visualization dan skeleton overlay

Spesialisasi & Production

  • Export model ke ONNX, verifikasi output konsisten, inference dengan ONNX Runtime
  • Quantization INT8 atau FP16, benchmark latency sebelum dan sesudah
  • Deploy ke edge device (Jetson atau RPi), benchmark FPS real-world
  • Serve model dengan Triton atau BentoML, konfigurasi dynamic batching
  • Setup monitoring Prometheus + Grafana untuk latency dan confidence drift
  • Implementasi active learning pipeline dengan uncertainty sampling
  • Anotasi menggunakan Label Studio atau CVAT, export ke format YOLO/COCO

Generative CV (Opsional tapi direkomendasikan)

  • Zero-shot classification dengan CLIP, image-text similarity search
  • Generate gambar dengan Stable Diffusion, eksperimen dengan CFG scale dan step count
  • Fine-tune SD dengan LoRA pada custom dataset (5–30 gambar)
  • Inpainting dan image editing dengan SD pipeline
  • Generate video pendek dengan AnimateDiff atau CogVideoX
