2.1 CNN untuk Vision // arsitektur yang merevolusi computer vision

Convolutional Neural Network (CNN) adalah arsitektur neural network yang dirancang khusus untuk data grid seperti gambar. Keunggulan utamanya: parameter sharing (filter yang sama dipakai di seluruh gambar) dan local connectivity (setiap neuron hanya terhubung ke area lokal). Ini membuat CNN jauh lebih efisien dan efektif daripada fully-connected network untuk vision.

Tiga komponen utama CNN

  • Conv layer: belajar filter/kernel secara otomatis dari data. Layer awal belajar tepi dan warna, layer dalam belajar fitur kompleks (mata, roda, dll)
  • Pooling layer: down-sampling — MaxPool atau AvgPool. Mengurangi ukuran spasial, meningkatkan receptive field, memberikan sedikit translasi-invariance
  • FC layer: di akhir jaringan, menggabungkan semua fitur untuk prediksi final. Sering digantikan Global Average Pooling (GAP) di arsitektur modern

Evolusi arsitektur CNN

  • AlexNet (2012): pertama dengan deep CNN + GPU, menang ImageNet
  • VGG (2014): deep dan sederhana, hanya conv 3×3
  • ResNet (2015): skip connections memungkinkan 100+ layers tanpa vanishing gradient
  • EfficientNet (2019): compound scaling depth/width/resolution secara seimbang
  • ConvNeXt (2022): CNN modern yang mencocoki ViT dengan redesign menyeluruh

import torch
import torch.nn as nn

class SimpleCNN(nn.Module):
    def __init__(self, num_classes: int = 10):
        super().__init__()
        self.features = nn.Sequential(
            # Block 1: 3→32 channels, 224→112
            nn.Conv2d(3, 32, kernel_size=3, padding=1),
            nn.BatchNorm2d(32),
            nn.ReLU(inplace=True),
            nn.MaxPool2d(2, 2),
            # Block 2: 32→64 channels, 112→56
            nn.Conv2d(32, 64, kernel_size=3, padding=1),
            nn.BatchNorm2d(64),
            nn.ReLU(inplace=True),
            nn.MaxPool2d(2, 2),
            # Block 3: 64→128 channels, 56→28
            nn.Conv2d(64, 128, kernel_size=3, padding=1),
            nn.BatchNorm2d(128),
            nn.ReLU(inplace=True),
            nn.MaxPool2d(2, 2),
        )
        self.classifier = nn.Sequential(
            nn.AdaptiveAvgPool2d((1, 1)),  # Global Avg Pool: (B,128,H,W) → (B,128,1,1)
            nn.Flatten(),
            nn.Dropout(0.3),
            nn.Linear(128, num_classes),
        )

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        return self.classifier(self.features(x))

# Training loop minimal
model = SimpleCNN(num_classes=10).cuda()
optimizer = torch.optim.AdamW(model.parameters(), lr=1e-3, weight_decay=1e-4)
criterion = nn.CrossEntropyLoss(label_smoothing=0.1)

for imgs, labels in dataloader:
    imgs, labels = imgs.cuda(), labels.cuda()
    optimizer.zero_grad()
    loss = criterion(model(imgs), labels)
    loss.backward()
    optimizer.step()
2.2 Transfer Learning & Fine-tuning // jangan train dari nol

Model yang dilatih pada ImageNet (1.2M gambar, 1000 kelas) sudah belajar fitur visual universal: tepi, tekstur, bentuk, objek. Transfer learning memanfaatkan ini — ambil backbone yang sudah terlatih, ganti head classifier, fine-tune untuk task kamu. Hasilnya jauh lebih baik daripada training dari nol, dengan data jauh lebih sedikit.

Strategi fine-tuning

  • Feature extraction: freeze semua layer, hanya train head baru. Cepat, butuh sedikit data. Cocok kalau domain mirip ImageNet
  • Partial fine-tuning: freeze layer awal (low-level features), unfreeze layer akhir + head. Balance antara kecepatan dan adaptasi
  • Full fine-tuning: unfreeze semua layer dengan learning rate lebih kecil. Butuh data lebih banyak, hasil terbaik jika domain sangat berbeda dari ImageNet

Learning rate scheduling

Layer-wise LR decay: layer lebih dalam (lebih dekat ke head) mendapat LR lebih besar, layer lebih dangkal mendapat LR sangat kecil. Ini melindungi pretrained features. Warmup: mulai dari LR sangat kecil, naikkan perlahan — mencegah shock pada pretrained weights di awal training. Cosine annealing: turunkan LR secara smooth hingga akhir training.

Pythontransfer_learning.py
import torch
import torchvision.models as models
from torch import nn

# Load pretrained ResNet50
backbone = models.resnet50(weights=models.ResNet50_Weights.IMAGENET1K_V2)

# Strategi 1: feature extraction — freeze backbone
for p in backbone.parameters():
    p.requires_grad = False

# Ganti head untuk num_classes kita
num_classes = 5
in_features = backbone.fc.in_features   # 2048
backbone.fc = nn.Sequential(
    nn.Dropout(0.3),
    nn.Linear(in_features, num_classes)
)
# Hanya head yang ter-train

# Strategi 2: partial fine-tuning — unfreeze layer4 + head
for p in backbone.parameters():
    p.requires_grad = False
for p in backbone.layer4.parameters():
    p.requires_grad = True
for p in backbone.fc.parameters():
    p.requires_grad = True

# Layer-wise LR decay (learning rate berbeda per layer group)
optimizer = torch.optim.AdamW([
    {'params': backbone.layer4.parameters(), 'lr': 1e-4},
    {'params': backbone.fc.parameters(), 'lr': 1e-3},
], weight_decay=1e-4)

# Cosine annealing + warmup
scheduler = torch.optim.lr_scheduler.OneCycleLR(
    optimizer, max_lr=[1e-4, 1e-3],
    steps_per_epoch=len(train_loader), epochs=20,
    pct_start=0.1  # 10% warmup
)

2.3 Vision Transformer (ViT) // attention mekanisme datang ke dunia vision

Vision Transformer (ViT, 2020) membuktikan bahwa arsitektur Transformer yang sukses di NLP bisa bekerja langsung untuk gambar — tanpa konvolusi sama sekali. Idenya: potong gambar menjadi patch kecil (16×16 piksel), flatten setiap patch menjadi vektor, lalu proses sebagai sequence token seperti kata dalam kalimat.

Cara ViT memproses gambar

  1. Gambar 224×224 dipotong menjadi patch 16×16 → 196 patch
  2. Setiap patch di-flatten dan di-project ke embedding dimension (mis. 768)
  3. Tambahkan positional embedding agar model tahu urutan spasial
  4. Tambahkan [CLS] token di depan sequence
  5. Proses dengan Transformer encoder (multi-head self-attention)
  6. Output [CLS] token dipakai untuk klasifikasi

ViT vs CNN: kapan pakai apa?

ViT unggul: data besar (>1M sampel), task butuh long-range dependencies (hubungan antara bagian gambar yang jauh), dan ketika pretrained pada dataset besar (ImageNet-21k, JFT). CNN unggul: data kecil-sedang, training dari nol, atau butuh efisiensi tinggi. Hybrid (CNN + Transformer): DeiT, Swin Transformer — best of both worlds.

2.4 Data Augmentasi // lebih banyak data dari data yang ada

Augmentasi data adalah teknik meningkatkan variasi data training dengan transformasi yang mempertahankan label. Hasilnya: model lebih robust terhadap variasi real-world (pencahayaan berbeda, angle berbeda, oklusi parsial). Ini adalah salah satu cara paling efektif meningkatkan generalisasi tanpa mengumpulkan data baru.

Augmentasi dasar vs advanced

Dasar: flip horizontal, rotasi kecil, brightness/contrast jitter, crop acak — hampir selalu safe untuk semua task.

Advanced: CutMix (gabung dua gambar + label), MixUp (blend dua gambar + soft label), RandAugment (pilih augmentasi secara random dari set besar), Mosaic (YOLO) — gabung 4 gambar. Hasilnya bisa signifikan tapi perlu tuning.

Albumentations vs torchvision

torchvision.transforms: simpel, terintegrasi, tapi lambat (CPU only). Albumentations: 4–10x lebih cepat, 70+ augmentasi, support joint transform untuk gambar + mask + bounding box (krusial untuk detection dan segmentation). Untuk production: selalu pakai Albumentations.

# CutMix — advanced, signifikan untuk ImageNet-scale training
def cutmix(imgs, labels, alpha=1.0):
    lam = np.random.beta(alpha, alpha)
    B = imgs.size(0)
    idx = torch.randperm(B)
    h, w = imgs.shape[-2:]
    cut_h = int(h * np.sqrt(1 - lam))
    cut_w = int(w * np.sqrt(1 - lam))
    cx, cy = np.random.randint(w), np.random.randint(h)
    x1, y1 = max(0,cx-cut_w//2), max(0,cy-cut_h//2)
    x2, y2 = min(w,cx+cut_w//2), min(h,cy+cut_h//2)
    imgs[:, :, y1:y2, x1:x2] = imgs[idx, :, y1:y2, x1:x2]
    lam = 1 - (x2-x1)*(y2-y1)/(h*w)
    return imgs, labels, labels[idx], lam
2.5 Loss Functions untuk Vision // sinyal training yang tepat per task

Loss function adalah sinyal yang mengatakan model seberapa salah prediksinya. Pilihan loss yang salah bisa membuat training gagal meski arsitektur dan data sudah benar. Setiap task vision punya karakteristik yang membutuhkan loss khusus.

Classification losses

  • CrossEntropyLoss: standard untuk multi-class, menggabungkan softmax + NLL
  • Label smoothing: distribusikan sedikit probabilitas ke kelas lain (0.1), mencegah overconfidence
  • Focal Loss: down-weight easy examples, fokus pada hard examples — krusial untuk class imbalance besar (mis. detection: banyak background)
  • Binary Cross-entropy: untuk multi-label (satu gambar bisa punya banyak kelas)

Regression dan segmentation losses

  • L1 / Smooth L1 (Huber): untuk bounding box regression, robust terhadap outlier
  • IoU Loss / GIoU / DIoU: loss berbasis overlap langsung untuk box regression — lebih baik dari L2 koordinat
  • Dice Loss: untuk segmentasi, mengoptimalkan F1/IoU langsung
  • BCE + Dice: kombinasi umum untuk segmentasi binary
Pythonvision_losses.py
import torch
import torch.nn as nn
import torch.nn.functional as F

# Focal Loss — untuk class imbalance besar
class FocalLoss(nn.Module):
    def __init__(self, alpha=1.0, gamma=2.0):
        super().__init__()
        self.alpha, self.gamma = alpha, gamma

    def forward(self, inputs, targets):
        ce = F.cross_entropy(inputs, targets, reduction='none')
        pt = torch.exp(-ce)
        return (self.alpha * (1 - pt)**self.gamma * ce).mean()

# Dice Loss — untuk segmentasi
class DiceLoss(nn.Module):
    def __init__(self, smooth=1.0):
        super().__init__()
        self.smooth = smooth

    def forward(self, pred, target):
        pred = torch.sigmoid(pred)
        intersection = (pred * target).sum(dim=(2,3))
        dice = (2 * intersection + self.smooth) / (pred.sum(dim=(2,3)) + target.sum(dim=(2,3)) + self.smooth)
        return 1 - dice.mean()

# Kombinasi BCE + Dice untuk segmentasi
bce = nn.BCEWithLogitsLoss()
dice = DiceLoss()
loss = 0.5 * bce(pred, target) + 0.5 * dice(pred, target)
2.6 Model Evaluation & Metrics // akurasi bukan satu-satunya ukuran

Akurasi saja tidak cukup untuk mengevaluasi model vision. Dataset imbalanced membuat akurasi menyesatkan — model yang selalu prediksi kelas mayoritas bisa dapat akurasi 95% tapi tidak berguna. Setiap task punya metrik yang lebih informatif.

Metrik classification

  • Accuracy: baik hanya untuk balanced dataset
  • Precision / Recall / F1: lebih informatif untuk imbalanced. Precision: dari yang diprediksi positif, berapa yang benar. Recall: dari yang sebenarnya positif, berapa yang terdeteksi
  • ROC-AUC: performance di semua threshold — threshold-agnostic
  • Confusion matrix: lihat error pattern per kelas
  • Top-5 accuracy: dipakai di ImageNet — apakah label benar ada di 5 prediksi teratas

Metrik detection & segmentation

  • IoU (Intersection over Union): overlap antara predicted dan ground truth box. IoU > 0.5 biasanya dianggap "benar"
  • mAP (mean Average Precision): metrik standar detection. mAP@0.5 atau mAP@[0.5:0.95]
  • Pixel Accuracy: fraction piksel yang benar (misleading jika imbalanced)
  • Mean IoU (mIoU): IoU per kelas kemudian di-average — standar segmentasi
Pythonevaluation.py
from sklearn.metrics import classification_report, confusion_matrix
import torch
import numpy as np

# Evaluasi classification
def evaluate_classifier(model, dataloader, device):
    model.eval()
    all_preds, all_labels = [], []
    with torch.no_grad():
        for imgs, labels in dataloader:
            preds = model(imgs.to(device)).argmax(dim=1).cpu()
            all_preds.extend(preds.numpy())
            all_labels.extend(labels.numpy())
    print(classification_report(all_labels, all_preds))
    return confusion_matrix(all_labels, all_preds)

# IoU untuk segmentasi
def compute_miou(pred_masks, gt_masks, num_classes):
    ious = []
    for cls in range(num_classes):
        intersection = ((pred_masks == cls) & (gt_masks == cls)).sum().item()
        union = ((pred_masks == cls) | (gt_masks == cls)).sum().item()
        if union > 0:
            ious.append(intersection / union)
    return np.mean(ious)
