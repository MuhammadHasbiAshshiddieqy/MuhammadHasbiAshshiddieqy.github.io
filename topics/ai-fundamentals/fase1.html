1.1 Linear Regression // garis terbaik melalui data

Algoritma ML paling sederhana: temukan garis lurus yang paling cocok menggambarkan hubungan antara dua variabel. Cocok untuk memprediksi nilai numerik kontinu.

Rumus: y = wx + b — model mencari nilai w (kemiringan) dan b (intercept) yang meminimalkan total error terhadap semua data.

Cara Kerja — Gradient Descent

Model tidak langsung "tahu" garis terbaik. Ia mulai dari garis acak, lalu perlahan-lahan menggeser bobot w dan b ke arah yang mengurangi error. Proses ini disebut Gradient Descent.

  • MSE (Mean Squared Error) — ukuran error: rata-rata kuadrat selisih prediksi vs kenyataan.
  • R² (R-squared) — seberapa baik model menjelaskan variasi data (0–1, makin dekat 1 makin baik).
  • Overfitting — model terlalu "hafal" data training, gagal di data baru. Solusi: lebih banyak data atau regularisasi.

Kapan digunakan: Prediksi harga rumah, estimasi penjualan, analisis tren.

1.2 Logistic Regression // klasifikasi dengan probabilitas

Nama "regression" sedikit menyesatkan — ini sebenarnya algoritma klasifikasi. Alih-alih garis lurus, ia menggunakan kurva berbentuk S (sigmoid) untuk memprediksi probabilitas suatu kelas: apakah email ini spam? apakah tumor ini ganas?

Output: Angka antara 0 dan 1 (probabilitas). Jika > 0.5 → kelas positif, jika ≤ 0.5 → kelas negatif.

Kelebihan & Kekurangan

  • Sederhana, cepat, mudah diinterpretasi
  • Menghasilkan probabilitas, bukan hanya label
  • Tanpa feature engineering tambahan, hanya bisa memisahkan kelas yang linearly separable
  • Tidak bisa menangkap pola kompleks non-linear

Kapan digunakan: Deteksi spam, prediksi churn pelanggan, diagnosis medis dasar, credit scoring.

1.3 Decision Tree // pohon pertanyaan ya/tidak

Decision Tree belajar dengan membuat serangkaian pertanyaan ya/tidak tentang fitur data, seperti bermain 20 Questions. Di setiap node, model memilih pertanyaan yang paling efektif memisahkan kelas-kelas yang ada.

Visualisasi Interaktif — Pohon Keputusan "Layak Kredit?"

Klik tombol untuk melihat bagaimana data diklasifikasikan

Konsep Penting

  • Gini Impurity / Entropy — ukuran seberapa "campur" data di sebuah node. Pertanyaan terbaik = yang paling mengurangi impurity.
  • Depth (kedalaman) — semakin dalam pohon, semakin "hafal" data training → risiko overfitting tinggi.
  • Pruning — memangkas cabang yang tidak berguna untuk mengurangi overfitting.

Kelebihan: Mudah dipahami manusia, tidak perlu normalisasi data, bisa handle data kategorikal. Kekurangan: Sangat rentan overfitting jika pohon terlalu dalam.

1.4 Random Forest // kekuatan kolektif pohon-pohon acak

Satu decision tree mudah overfitting. Solusinya? Bangun ratusan pohon dengan data dan fitur yang berbeda-beda (acak), lalu ambil suara terbanyak (majority voting). Inilah Random Forest.

Analogi: Jika kamu ingin nasihat penting, apakah kamu tanya satu orang saja, atau tanya 100 orang berbeda dan ikuti pendapat mayoritas? Kolektif lebih andal dari individu.

Mengapa Lebih Baik dari Single Tree?

  • Bagging (Bootstrap Aggregating) — tiap pohon dilatih pada subset data acak, mengurangi variance.
  • Feature randomness — tiap split hanya mempertimbangkan subset fitur acak, memastikan diversitas pohon.
  • Out-of-bag error — data yang tidak dipakai tiap pohon bisa digunakan sebagai validation gratis.

Kapan digunakan: Hampir semua masalah tabular (data berbentuk tabel). Sangat populer di kompetisi data science. Tahan terhadap outlier dan missing values.

1.5 Support Vector Machine (SVM) // garis pemisah dengan margin terlebar

SVM tidak hanya mencari sembarang garis yang memisahkan dua kelas — ia mencari garis dengan margin terlebar antara kelas-kelas tersebut. Semakin lebar margin, semakin "yakin" model dalam prediksinya.

Yang membuat SVM istimewa: dengan kernel trick, ia bisa memisahkan data yang tidak bisa dipisahkan oleh garis lurus — dengan cara "memproyeksikan" data ke dimensi yang lebih tinggi.

Visualisasi Interaktif — Maximum Margin Classifier

Garis kuning = hyperplane, daerah abu = margin, titik dilingkari = support vectors

Kernel Trick — Sihir SVM

  • Linear kernel — separasi langsung di dimensi asli. Cepat, baik untuk data high-dimensional seperti teks.
  • RBF (Radial Basis Function) — memproyeksikan ke dimensi tak terbatas. Bisa memisahkan data dengan batas non-linear kompleks.
  • Support Vectors — titik-titik data yang paling dekat dengan hyperplane. Hanya ini yang menentukan posisi garis pemisah.

Kapan digunakan: Klasifikasi teks (dengan linear kernel), pengenalan gambar tradisional, dataset ukuran sedang dengan batas keputusan kompleks.
