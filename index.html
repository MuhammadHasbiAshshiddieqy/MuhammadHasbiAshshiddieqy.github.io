Glosarium Agentic AI

Istilah-istilah teknis yang digunakan di seluruh panduan ini, dengan penjelasan singkat dan konteks penggunaannya dalam agent engineering.

Istilah
Penjelasan
A
Agent
Sistem yang menggabungkan LLM dengan tools dan execution loop — dapat mengambil tindakan berulang sampai task selesai. Berbeda dari chatbot yang hanya menjawab satu pertanyaan.
Agentic Loop
Pola eksekusi berulang: LLM reason → tool call → observe result → repeat. Loop ini yang membedakan agent dari one-shot LLM call.
Attention Mechanism
Mekanisme di Transformer yang memungkinkan setiap token "memperhatikan" token lain dalam context window. Dasar dari kemampuan reasoning LLM.
Asyncio
Framework Python untuk concurrent programming tanpa multi-threading. Penting untuk agent yang memanggil banyak tools secara paralel. Gunakan async/await dan asyncio.gather().
B
BPE (Byte-Pair Encoding)
Algoritma tokenisasi yang memecah teks menjadi subword units. "tokenization" → ["token", "ization"]. Kata bahasa Indonesia sering terpecah lebih banyak daripada bahasa Inggris — ini mempengaruhi jumlah token dan biaya.
Backpressure
Mekanisme sinyal "sistem penuh, kurangi input" dari downstream ke upstream. Penting agar agent tidak membanjiri tool atau database dengan request melebihi kapasitas.
C
Context Window
"Working memory" LLM — semua teks (system prompt, history, tool results) yang bisa dilihat model saat satu inference. Claude memiliki context window hingga ratusan ribu token.
CoT (Chain-of-Thought)
Teknik prompting yang mendorong model untuk "berpikir langkah demi langkah" sebelum menjawab. Meningkatkan akurasi untuk masalah multi-step. Bisa explicit ("think step by step") atau implicit.
Checkpointing
Menyimpan state agent (conversation history, tool results, progress) ke storage persisten. Memungkinkan agent resume jika proses crash atau di-restart, tanpa harus mulai dari awal.
E
Embedding
Representasi teks sebagai vektor angka berdimensi tinggi. Teks yang semantically similar menghasilkan vektor yang berdekatan. Dasar dari semantic search dan RAG.
Emergent Behavior
Kemampuan yang muncul dari LLM berskala besar tanpa diprogram secara eksplisit — seperti reasoning, analogy, dan code generation. Bukan hasil symbolic programming, tapi pola statistik dari training data.
F
Fan-Out / Fan-In
Pola distribusi kerja: Fan-out = satu task dibagi ke banyak subagent paralel. Fan-in = hasil dikumpulkan dan digabungkan. Efektif untuk tasks yang bisa diparalelisasi (research, batch processing).
Finish Reason
Alasan model berhenti generate. end_turn = selesai natural. tool_use = perlu tool (Anthropic). tool_calls = perlu tool (OpenAI). max_tokens = terpotong. Harness wajib cek ini setiap iterasi.
G
Guardrails
Lapisan keamanan yang membatasi tindakan agent — input validation, output filtering, tool allowlist, human-in-the-loop untuk irreversible actions. Tidak boleh ditambahkan belakangan — harus dirancang dari awal.
H
Hallucination
LLM menghasilkan informasi yang terdengar meyakinkan tapi salah atau tidak ada. Konsekuensi dari sifat probabilistik next-token prediction. Atasi dengan tools (grounding ke data nyata) dan verifikasi.
Harness
Kode non-LLM yang mengorkestrasi agent: routing tool calls, injecting results, managing history, detecting stopping condition, handling errors. "Pengemudi" yang menggunakan LLM sebagai "mesin".
HNSW
Hierarchical Navigable Small World — algoritma approximate nearest neighbor search yang digunakan di vector database (Chroma, Qdrant). Cepat tapi approximate (hasilnya bukan pasti yang paling mirip). Alternatif: IVFFlat (lebih eksak, lebih lambat).
Human-in-the-Loop
Pattern di mana agent berhenti dan meminta konfirmasi manusia sebelum mengambil tindakan yang irreversible atau berisiko tinggi. Wajib untuk actions seperti delete, send email, atau financial transactions.
I – K
IVFFlat
Inverted File Index — algoritma vector search yang lebih eksak dari HNSW. Dibagi ke cluster dulu, lalu search dalam cluster. Lebih lambat tapi lebih akurat. Digunakan di pgvector (PostgreSQL).
KV Cache
Key-Value Cache — menyimpan computed attention key-value pairs dari tokens yang sudah diproses. Accelerates inference untuk pesan yang berulang. Prompt caching Anthropic memanfaatkan ini — token yang sama tidak perlu diproses ulang.
L – M
Langfuse
Platform open-source observability untuk LLM/agent. Fitur: trace visualization, token cost tracking, eval scoring. Bisa self-host atau gunakan cloud. Alternatif: LangSmith (LangChain), Helicone.
MCP (Model Context Protocol)
Protokol standar terbuka (Anthropic) untuk menghubungkan LLM dengan tools dan data sources. Memisahkan tool server dari model server — tools bisa di-reuse lintas model dan platform.
Multi-Agent
Arsitektur dengan beberapa agent yang bekerja bersama. Orchestrator mendelegasikan ke subagent yang spesialis. Cocok untuk tasks kompleks yang bisa dibagi berdasarkan domain atau paralelisasi.
O – P
Orchestrator
Agent tingkat tinggi yang merencanakan dan mendelegasikan ke subagent. Bertanggung jawab atas decomposition task, routing ke specialist, dan agregasi hasil akhir.
Prompt Injection
Serangan di mana konten eksternal (dari web scrape, file, atau user input) menyisipkan instruksi yang mengubah perilaku agent. Misalnya dokumen yang berisi "ignore previous instructions". Wajib di-mitigasi di production.
R
RAG (Retrieval-Augmented Generation)
Teknik menggabungkan vector search dengan LLM: query → semantic search → retrieve relevant chunks → inject ke context → generate. Memungkinkan LLM menjawab berdasarkan knowledge base custom tanpa fine-tuning.
ReAct Pattern
Reasoning + Acting — pola di mana agent bergantian: "think" (reasoning step) dan "act" (tool call). Membuat decision-making agent lebih transparan dan lebih mudah di-debug dibanding langsung action.
RPM / TPM
Requests Per Minute / Tokens Per Minute — batas API dari provider. Anthropic memberlakukan keduanya per API key. Harness production harus track usage dan implement backoff saat mendekati limit.
S – T
SentencePiece
Library tokenisasi Google yang digunakan oleh beberapa model (Gemini, LLaMA). Mirip BPE tapi bekerja pada karakter/byte level, lebih robust untuk bahasa non-Latin.
Stop Reason
Lihat: Finish Reason.
Temperature
Parameter yang mengontrol "keacakan" output (0.0 = deterministik, 2.0 = sangat random). Untuk agent production: gunakan 0.0–0.3 untuk konsistensi. Untuk creative tasks: 0.7–1.0.
Token
Unit terkecil yang diproses LLM — bisa satu kata, satu suku kata, atau satu karakter. "tokenization" ≈ 3-4 tokens. "gajah" ≈ 2-3 tokens. Semua biaya dan limit dihitung dalam token.
ToT (Tree of Thought)
Ekstensi CoT yang mengeksplorasi beberapa reasoning paths sekaligus (tree), bukan satu linear chain. Lebih efektif untuk problems yang butuh backtracking, tapi jauh lebih mahal secara token.
Tool Call / Tool Use
Mekanisme di mana LLM tidak langsung menjawab tapi menghasilkan structured request untuk menjalankan fungsi eksternal. Model tidak menjalankan tool sendiri — harness yang mengeksekusi dan mengembalikan hasilnya.
Trajectory
Urutan langkah yang diambil agent untuk menyelesaikan task: thought → action → observation → thought → ... Trajectory eval menganalisis apakah urutan ini efisien dan logis, bukan hanya apakah hasilnya benar.
V – X
Vector Database
Database yang dioptimalkan untuk menyimpan dan mencari embedding vectors. Mendukung similarity search (nearest neighbor). Contoh: Chroma (lokal/mudah), Qdrant (production), pgvector (PostgreSQL extension).
Xvfb
X Virtual Framebuffer — virtual display server di Linux yang memungkinkan aplikasi GUI berjalan tanpa monitor fisik. Dibutuhkan untuk computer use agent di server headless.
