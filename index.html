Glosarium Agentic AI

Istilah-istilah teknis yang digunakan di seluruh panduan ini, dengan penjelasan singkat dan konteks penggunaannya dalam agent engineering.

Istilah
Penjelasan
A
Agent
Sistem yang menggabungkan LLM dengan tools dan execution loop — dapat mengambil tindakan berulang sampai task selesai. Berbeda dari chatbot yang hanya menjawab satu pertanyaan.
Agentic Loop
Pola eksekusi berulang: LLM reason → tool call → observe result → repeat. Loop ini yang membedakan agent dari one-shot LLM call.
Attention Mechanism
Mekanisme di Transformer yang memungkinkan setiap token "memperhatikan" token lain dalam context window. Dasar dari kemampuan reasoning LLM.
Asyncio
Framework Python untuk concurrent programming tanpa multi-threading. Penting untuk agent yang memanggil banyak tools secara paralel. Gunakan async/await dan asyncio.gather().
B
BPE (Byte-Pair Encoding)
Algoritma tokenisasi yang memecah teks menjadi subword units. "tokenization" → ["token", "ization"]. Kata bahasa Indonesia sering terpecah lebih banyak daripada bahasa Inggris — ini mempengaruhi jumlah token dan biaya.
Backpressure
Mekanisme sinyal "sistem penuh, kurangi input" dari downstream ke upstream. Penting agar agent tidak membanjiri tool atau database dengan request melebihi kapasitas.
C
Context Window
"Working memory" LLM — semua teks (system prompt, history, tool results) yang bisa dilihat model saat satu inference. Claude memiliki context window hingga ratusan ribu token.
CoT (Chain-of-Thought)
Teknik prompting yang mendorong model untuk "berpikir langkah demi langkah" sebelum menjawab. Meningkatkan akurasi untuk masalah multi-step.
Checkpointing
Menyimpan state agent ke storage persisten. Memungkinkan agent resume jika proses crash atau di-restart.
E
Embedding
Representasi teks sebagai vektor angka berdimensi tinggi. Teks yang semantically similar menghasilkan vektor yang berdekatan. Dasar dari semantic search dan RAG.
Emergent Behavior
Kemampuan yang muncul dari LLM berskala besar tanpa diprogram secara eksplisit — reasoning, analogy, code generation.
F
Fan-Out / Fan-In
Pola distribusi kerja: Fan-out = satu task dibagi ke banyak subagent paralel. Fan-in = hasil dikumpulkan dan digabungkan.
Finish Reason
Alasan model berhenti generate. end_turn = selesai natural. tool_use = perlu tool (Anthropic). max_tokens = terpotong.
G
Guardrails
Lapisan keamanan yang membatasi tindakan agent — input validation, output filtering, tool allowlist, human-in-the-loop untuk irreversible actions.
H
Hallucination
LLM menghasilkan informasi yang terdengar meyakinkan tapi salah. Atasi dengan tools (grounding ke data nyata) dan verifikasi.
Harness
Kode non-LLM yang mengorkestrasi agent: routing tool calls, injecting results, managing history, detecting stopping condition, handling errors.
HNSW
Hierarchical Navigable Small World — algoritma approximate nearest neighbor search. Cepat tapi approximate.
Human-in-the-Loop
Pattern di mana agent berhenti dan meminta konfirmasi manusia sebelum tindakan irreversible atau berisiko tinggi.
I – K
KV Cache
Key-Value Cache — menyimpan computed attention key-value pairs. Prompt caching Anthropic memanfaatkan ini — token yang sama tidak perlu diproses ulang.
L – M
MCP (Model Context Protocol)
Protokol standar terbuka (Anthropic) untuk menghubungkan LLM dengan tools dan data sources. Tools bisa di-reuse lintas model dan platform.
Multi-Agent
Arsitektur dengan beberapa agent yang bekerja bersama. Orchestrator mendelegasikan ke subagent yang spesialis.
O – P
Orchestrator
Agent tingkat tinggi yang merencanakan dan mendelegasikan ke subagent.
Prompt Injection
Serangan di mana konten eksternal menyisipkan instruksi yang mengubah perilaku agent. Wajib di-mitigasi di production.
R
RAG (Retrieval-Augmented Generation)
Menggabungkan vector search dengan LLM: query → semantic search → retrieve → inject ke context → generate.
ReAct Pattern
Reasoning + Acting — agent bergantian "think" dan "act". Decision-making lebih transparan dan mudah di-debug.
S – T
Temperature
Parameter yang mengontrol keacakan output via sampling distribution. Ikuti rekomendasi provider untuk model yang digunakan.
Token
Unit terkecil yang diproses LLM. Semua biaya dan limit dihitung dalam token.
Tool Call / Tool Use
Mekanisme LLM menghasilkan structured request untuk menjalankan fungsi eksternal. Harness yang mengeksekusi dan mengembalikan hasilnya.
V
Vector Database
Database yang dioptimalkan untuk menyimpan dan mencari embedding vectors. Contoh: Chroma, Qdrant, pgvector.
Glosarium ini akan terus diperbarui.
